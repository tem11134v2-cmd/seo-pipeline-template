# Architecture Decision Records

Каждый ADR описывает одно крупное архитектурное решение: контекст, что выбрали, альтернативы, почему отвергли, последствия.

Это не «как пользоваться», это **почему оно устроено так**. Если ты — будущая сессия Claude или новый разработчик системы, читай ADR прежде чем менять архитектуру.

## Как добавить новый ADR

1. Создать `docs/adr/NNN-кратко-о-решении.md` по шаблону `_template.md`.
2. Записать в таблицу выше.
3. Если ADR заменяет старое решение — пометить старое статусом «Заменено (см. NNN)» и добавить ссылку.
