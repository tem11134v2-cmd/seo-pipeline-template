# ADR-011: Самозащита шаблона от запуска клиентских команд

**Статус:** Принято

**Дата:** 2026-05-31

## Контекст

Каталог `seo-pipeline-template` - это **шаблон**, который клонируется в каждый клиентский проект (`~/seo-projects/<slug>/`) через `/new-project`. В нём ведётся разработка машинерии (скилы, агенты, хуки, скрипты), но он не должен содержать клиентских данных.

Реальный инцидент: `/handoff-process` был по ошибке запущен в каталоге шаблона (вместо клиентского проекта) и применил клиента traffic-masters прямо в шаблон - `ЗАКАЗЧИК.md` + `template.html` оказались в корне шаблона. Поскольку шаблон клонируется, **каждый будущий клиент унаследовал бы traffic-masters**.

CLAUDE.md содержит правило «не запускай клиентские скилы в шаблоне», но это **рекомендация**, не барьер - модель или пользователь могут ошибиться. Нужен технический барьер, при этом он **не должен задеть клиентские клоны** (там команды обязаны работать).

## Решение

Два рубежа, оба завязаны на untracked-маркер `.claude/.is-template-root`:

1. **Upfront-барьер (UserPromptSubmit-хук `guard-template-root.sh`).** Если маркер есть, блокирует клиентские slash-команды до запуска (`exit 2` + stderr). Разрешены `/guide` и `/sync-from-template`; обычные (не slash) промпты не трогаются. Fail-open: любая внутренняя ошибка хука -> `exit 0` (не ломаем сессию).

2. **Backstop (pre-commit).** Если маркер есть, запрещает КОММИТ клиентских артефактов (`ЗАКАЗЧИК.md`, `template.html`, `topics.xlsx`, `articles/`, `analyses/`, `structures/`, `strategies/`, `topics/`). Удаления разрешены (`--diff-filter=ACMR`), чтобы можно было вычищать уже просочившееся. Это страховка на случай, если первый рубеж как-то обойдён.

**Ключевое свойство - fail-safe через untracked-маркер.** Маркер в `.gitignore`, то есть не коммитится. `git clone` переносит только закоммиченную историю, поэтому клиентские клоны маркер НЕ получают -> у них оба рубежа неактивны и команды работают как обычно. Отсутствие маркера = разрешено.

## Альтернативы

- **Детект по git remote.** Отвергнуто: клиентские клоны наследуют тот же `origin` (`seo-pipeline-template.git`) - проверено на `traffic_masters_ru` и `mamapro_ru`. Не отличить шаблон от клиента.
- **Tracked-маркер + удаление в `/new-project`.** Отвергнуто как fail-unsafe: если `/new-project` не удалит маркер, клиент окажется полностью заблокирован (все команды не работают). Untracked-маркер фейлит в безопасную сторону.
- **Только проверка в «шаге 0» скилов.** Отвергнуто: рекомендация, а не барьер; не ловит прямой `/handoff-process`.
- **PreToolUse на инструменте `Skill`.** Отвергнуто как менее надёжное: пользовательская slash-команда может не проходить через инструмент `Skill`, а `UserPromptSubmit` видит сырой промпт всегда.

## Последствия

- В шаблоне клиентские команды отключены, а коммит клиентских артефактов отклоняется. Разработка машинерии (Edit/Write/Bash, `/guide`, `/sync-from-template`) - без ограничений.
- Клиентские проекты не затронуты (нет маркера). Хук и backstop у них присутствуют (committed), но неактивны.
- Свежий dev-каталог шаблона на другой машине защиты не получит автоматически - маркер `.claude/.is-template-root` нужно создать вручную (содержимое не важно). Это осознанный размен в пользу fail-safe для клиентов.
- Снять защиту в самом шаблоне временно - удалить маркер.
