# ADR-029: Скил /custom-question - вопрос заказчика через гейт трактовки

**Статус:** Принято

**Дата:** 2026-07-16

## Контекст

Заказчики задают нестандартные вопросы и поднимают нестыковки, которые не ложатся
ни в один скил конвейера. Ответ «на ходу» рисковал сразу двумя вещами: неверно
понять сам вопрос (решить не то) и утечь клиенту SEO-жаргон. Плюс не было памяти
решенных кейсов - каждый похожий вопрос разбирался с нуля.

## Решение

**Новый worktree-скил `/custom-question` с обязательным гейтом трактовки, решением
без жаргона и памятью в QA-журнале.**

- **Скил `/custom-question`** `[вопрос|файл] [--resume] [--format
  auto|answer|recommendation|doc]`; папка `questions/NNN-<slug>`; state machine
  `init -> context-done -> clarified|awaiting-client -> drafted -> verified ->
  delivered -> completed`.
- **context-gatherer (opus):** контекст по файлам проекта (ЗАКАЗЧИК.md, intake,
  meta задач, QA-журнал) + 2-4 трактовки вопроса; большая фактура - путями.
- **Обязательный гейт трактовки:** один раунд `AskUserQuestion` (трактовка +
  недостающие вводные, у каждого вопроса опция «передать проджекту») - при
  передаче создается `client_questions.md` человеческим языком, state
  `awaiting-client`.
- **solution-writer (opus):** решение в выбранном формате, клиентский текст без
  SEO-жаргона, обязательный блок «Что я не проверял».
- **solution-verifier (opus):** факт-чек по файлам проекта, простота языка,
  полнота, честность границ; лимит фиксов 2 (паттерн [ADR-025](025-final-verifiers.md)).
- **Память:** `QA-ЖУРНАЛ.md` (общий файл проекта, записи сверху) через
  `request-shared-edit`; `kb_upsert` только по явной просьбе; в main скил
  отказывает (только worktree, см. [ADR-001](001-worktree-first.md)).
- **pre-commit-бэкстоп** дополнен `questions/`+`QA-ЖУРНАЛ` и закрыт старый пробел
  (`audits/texts/faq`).

## Альтернативы

- **Отвечать на ходу в чате.** Отвергнуто: без гейта трактовки - риск решить не тот
  вопрос; нет заслона жаргона; нет памяти.
- **Автоматически выбрать одну трактовку.** Отвергнуто: нестандартные вопросы
  неоднозначны; один обязательный раунд `AskUserQuestion` (с опцией «передать
  проджекту») дешевле неверного ответа.
- **Всегда писать в kb.** Отвергнуто: kb - глобальная и кураторская;
  `QA-ЖУРНАЛ.md` (локальный по проекту) - память по умолчанию, kb только по
  просьбе.

## Последствия

**Хорошо:** неоднозначный вопрос клиента получает структурный ответ без жаргона с
явной трактовкой и границами; решенные кейсы копятся в `QA-ЖУРНАЛ.md`; верификатор
держит честность.
**Плохо:** новая поверхность (агенты + state machine); гейт трактовки добавляет
раунд с пользователем (это намеренно); счетчики README пришлось выправить
(28 скилов / 58 агентов, устранен дрейф «46»).
