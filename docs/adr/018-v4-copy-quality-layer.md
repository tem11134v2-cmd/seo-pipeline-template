# ADR-018: Слой качества копирайта v4 (смысловое = выбор, FACTS, стоп-листы + копи-валидатор)

**Статус:** Принято

**Дата:** 2026-06-08

## Контекст

Постмортемы 4 боевых проектов (синтез заказчика) показали: стратегия/структура стабильны, **вся масса итераций уходит в копирайтинг**. Два корня:
- **Root B - модель решает за собственника.** Смысловые решения (метафора, подача цены, CTA) подавались одним вариантом -> заказчик менял -> каскадная пересборка (смена метафоры = переделка 7 блоков).
- **Root A - текст «по максимуму, потом вычитание»** (жаргон, дубли, все цифры) + факты не зафиксированы единым источником («плавающая» гарантия, расхождения имён, арифметика на лету).

Ключевая поправка заказчика: **часть v4-правил - это экономия, вынужденная отсутствием субагентов в claude.ai** (минимизировать текст, «1 страница за прогон»). На Claude Code Desktop с веером субагентов мы это снимаем и **тратим компьют** (больше вариантов, отдельные проходы-валидаторы, параллель), а смысловые правила качества берём полностью.

## Решение

### 1. Смысловое = ВЫБОР заказчика (Root B)
`offer-strategist` для смысловых решений (`positioning`, `idea`/метафора +обязательный «без метафоры», `price_presentation`, `cta_tone`) выдаёт **2-4 варианта + `recommended` + обоснование**, `chosen:null`. Технические (стадия, формула, палитра) решает сам. `Analysis.docx` рендерит раздел «Решения на ВАШ выбор»; **существующий клиентский гейт** (ADR-015) фиксирует выбор в `decisions.<key>.chosen`; `page-writer` берёт `variants[chosen ?? recommended]`. Структурно убирает каскадные переделки. Регион-тест: вариант «как название западной бизнес-книги» помечается `region_flag` (региональному заказчику не зайдёт).

### 2. FACTS - единый источник истины (Root A, факты)
`facts.json` (реквизиты/гарантия/числа + `publish`-флаги + `opsec_restricted`) создаётся на сетапе. **Все цифры сайта - ТОЛЬКО оттуда**; offer-strategist/page-writer/faq-builder тянут числа из него. Изменение факта - правка в карточке, не по месту. Запрет арифметики на лету (NPS->%, годовой÷12).

### 3. Стоп-листы v4 + копи-валидатор (мы можем позволить отдельный проход)
- `COPY.md` дополнен слоем v4: жаргон маркетолога (JTBD/CJM/УТП в клиентском тексте), аббревиатуры в Hero=0, дворовая лексика, манипуляции + **тест ФАС**, стандарт плейсхолдеров без сленга, **OPSEC** для деликатных ниш, и **13-пунктовый чек-лист** + H1-правила (<=60, без метафоры/числа-фильтра).
- `verify-copy.mjs` - **механический** шлюз по page.json ДО сборки HTML (самозащита/жаргон/манипуляции/сленг/H1-длина/эмодзи; exit 0/2).
- `copy-auditor` - **агент-проход** «свежими глазами» (аналог `text-auditor` статей): прогоняет полные 13 пунктов вкл. семантику (за счёт/H2-выгода/сверка чисел с FACTS/карта повторений/OPSEC) и **чинит** page.json. Это замена claude.ai-ручного чек-листа на автоматический gate + выделенный субагент - **потому что на Claude Code это по карману**.

Где claude.ai заставлял «писать минимум, чтобы меньше вычищать», у нас page-writer пишет хорошо, а `copy-auditor` ловит балласт отдельным проходом. «1 страница за прогон» (claude.ai-ограничение) не переносим - оставляем параллельный веер.

### 4. Анти-перегруз + плейсхолдеры
`page-writer`: бюджет <=12 секций, блок только при якоре боли/CJM, 1 CTA/блок, карта повторений (факт <=1-2x), H1 <=60. Сленг-плейсхолдеры `[... - Ща Докрутим]` заменены на `[ЗАПОЛНИТЬ]`/«требует уточнения» во всём ките (был прямой v4-нарушитель в нашем коде).

## Альтернативы
- **Модель выбирает смысловое сама** (как было). Отвергнуто - корень дорогих переделок (Root B).
- **Минимизировать текст, чтобы меньше вычищать** (v4-экономия). Отвергнуто как claude.ai-вынужденность; вместо неё - отдельный проход-аудитор (компьют дешевле итераций с клиентом).
- **Валидатор только скриптом.** Скрипт ловит механику; семантику (тон, переформулировка самозащиты, «за счёт») ловит только LLM-проход. Поэтому оба: `verify-copy.mjs` + `copy-auditor`.
- **FACTS как часть inputs.json.** Вынесли отдельно - это переходящий между всеми агентами источник, важно его выделить.

## Последствия
**Хорошо:** убирает корень дорогих копи-итераций (смысловое = выбор); цифры перестают «плавать» (FACTS); язык чистится гарантированным проходом; `settings.json`/pre-commit не меняются (verify - скрипт, audit - агент в веере). Заказчик видит выбор -> владеет брендом.
**Плохо:** +1 агент (`copy-auditor`) и +1 проход на страницу (стоимость компьюта - сознательно, ради меньшего числа клиент-итераций). `decisions`-схема усложняет strategy.json (митигация: backward-compat в docx-рендере + `chosen ?? recommended`). Гейт теперь требует от заказчика выбор по 4 пунктам (а не просто «ок») - это и есть цель.
