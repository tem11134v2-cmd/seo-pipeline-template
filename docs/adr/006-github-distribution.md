# ADR-006: Шаблон распространяется как публичный GitHub-репо

**Статус:** Принято

**Дата:** 2026-05-25

## Контекст

Изначально шаблон проекта (`template-project`) жил локально на одной машине пользователя. Скил `/setup-project` (старая версия) копировал папку шаблона в `~/seo-projects/<client>/` через `cp -r`.

Это работало пока одна машина. Проблемы появлялись когда:
- Пользователь хочет работать на двух машинах (домашняя + рабочая) — шаблон не синхронизируется.
- Появляются улучшения системы (новые скилы, исправления) — нужен механизм обновления.
- Кто-то ещё хочет использовать систему (друг, заказчик) — нет способа поделиться без zip-архивов.

## Решение

Template-project опубликован как **публичный GitHub-репо**: `https://github.com/tem11134v2-cmd/seo-pipeline-template`.

Установка нового клиента (вместо старого `/setup-project <URL>`):
1. Пользователь копирует мини-промт из README в новую сессию Claude Code Desktop.
2. Промт делает `git clone https://github.com/.../seo-pipeline-template.git ~/seo-projects/<slug>/`.
3. Промт делает `npm install` и `git config core.hooksPath .claude/git-hooks`.
4. Пользователь открывает созданную папку в новой worktree-сессии и запускает `/setup-project <URL клиента>` (это уже другой скил — исследование сайта).

Старый скил `/setup-project` (копирующий) **удалён**. Его роль (создание проекта) распилена:
- Клонирование — мини-промт.
- Исследование сайта — новый `/setup-project` в worktree (использует тех же агентов client-profiler + template-designer).

## Альтернативы

### A. Оставить локальный template-project

Не публиковать никуда. Отвергнуто: пользователь явно сказал, что хочет GitHub для синхронизации и тиражирования.

### B. Приватный GitHub-репо

Только для пользователя. Отвергнуто: клонирование требует auth, лишний шаг. Содержимое (методология) и так в `~/.claude/seo-knowledge/`, который отдельно — публичный шаблон ничего секретного не содержит.

### C. GitHub-template-репо (с кнопкой «Use this template»)

GitHub имеет фичу «Template repository». При клике «Use this template» создаётся новое репо у пользователя. Отвергнуто (пока): фича сделана для веб-разработки (новый репо), у нас нет цели создавать отдельный репо для каждого клиента — клиент это локальная папка, не репо на GitHub. Может быть пересмотрено в будущем.

### D. NPM-пакет

`npm install -g seo-pipeline-template` создаёт папку при вызове CLI. Отвергнуто: оверкилл, нет необходимости публиковать в npm registry, плюс это новый процесс упаковки/публикации каждой версии.

## Последствия

**Хорошо:**
- Установка на любой машине: открыл Claude Code Desktop, скопировал промт, всё.
- Обновления: `git pull` в template-project, новые клиенты получают свежую версию.
- Открытый исходник: можно делиться, принимать вклад, форкать.
- Версионность через git tags (если понадобится).

**Плохо:**
- Существующие клиенты (склонированные раньше) **не получают обновления автоматически**. Чтобы обновить — нужно `git pull origin main` в папке клиента. Это **не безопасно** для активных клиентов (мерж нашего main в клиентский main может затронуть локальные изменения).
  - Решение на будущее: добавить скил `/update-template` который умеет аккуратно перенести только обновления `.claude/` без затрагивания клиентских файлов.
- README с мини-промтом нужно поддерживать актуальным — если структура изменится, промт сломается у новых пользователей.
