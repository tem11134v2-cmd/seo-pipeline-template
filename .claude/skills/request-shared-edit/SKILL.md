# request-shared-edit

Worktree-сессия не может править общие файлы напрямую (pre-commit hook откажет). Этот скил создаёт **отложенный запрос** на правку, который применится позже в main.

## Когда использовать

- Заметил опечатку в `ЗАКАЗЧИК.md` (например, неправильный код региона)
- Нужно добавить ссылку в секцию «Перелинковка» ЗАКАЗЧИК.md
- Нужно поправить стиль в `template.html`
- Нужно дополнить `topics.xlsx` темой, которую только что придумал
- Любая другая правка вне папки текущей задачи

## Когда НЕ использовать

- Правка внутри своей задачи (`articles/NNN/...`) — это обычный Edit, никаких запросов не нужно.
- Если ты в main-сессии — просто правь напрямую.

## Аргументы

```
/request-shared-edit "<краткое описание правки>"
```

Если описание не передано — спросить у пользователя что именно поменять.

## Алгоритм

### 0. Проверка: мы в worktree?

```bash
GIT_DIR=$(git rev-parse --git-dir)
COMMON_DIR=$(git rev-parse --git-common-dir)
```

Если `GIT_DIR == COMMON_DIR` — мы в main, запрос не нужен. Сообщить:
> «Текущая сессия в main. Применяй правку напрямую через Edit.»

### 1. Сформировать промт-запрос

Создать файл `.claude/handoff-requests/<timestamp>-<task-slug>.md`:

```markdown
# Handoff request

- **Created:** <ISO timestamp UTC>
- **From task:** <task-slug, путь из current-task.txt>
- **From worktree:** <путь к worktree>
- **From branch:** <имя ветки>

## Запрос

<краткое описание из аргумента>

## Контекст

<развёрнутое объяснение: почему это нужно, что именно изменить, какие
поля/строки/секции файла затронуть, какие риски (например, может сломать
что-то другое), желаемый результат>

## Затрагиваемые файлы

- `ЗАКАЗЧИК.md` (секция «Перелинковка», добавить строку)
- `template.html` (опционально, если есть)
- `topics.xlsx` (опционально)

## Как проверить применение

<краткий чеклист: что должно измениться в файле / что должно перестать или
начать работать после правки>
```

Если описание короткое — заполнить разумные дефолты. Если непонятно из описания, что именно править — спросить пользователя:
> «Какой файл правим? Какую секцию/строку?»

### 2. Закоммитить запрос

```bash
git add .claude/handoff-requests/<file>
git commit -m "Handoff request: <короткое описание>"
```

Pre-commit hook разрешит — `.claude/handoff-requests/` в белом списке.

### 3. Сообщить пользователю

```
═══ ЗАПРОС СОХРАНЁН ═══
Файл: .claude/handoff-requests/<filename>.md
Запрос: <описание>

Применится автоматически:
  /handoff (в этой сессии)   → отнесёт файл в main вместе с merge
  /handoff-process (в main)  → применит правку к общему файлу
═══════════════════════
```

## Запреты

- Не пытайся применить правку из worktree напрямую через Edit ЗАКАЗЧИК.md — pre-commit hook откажет в коммите, и ты получишь только конфликт.
- Не редактируй уже созданные handoff-requests (они часть истории).
- Не создавай несколько одинаковых запросов — проверь существующие в `.claude/handoff-requests/` сначала.
