# MCP-карта для /seo-metategi

> Какие MCP-инструменты на каком этапе. Принцип: дорогую частотность/коммерциализацию батчим ОДИН раз на весь проект (Фаза 1), per-page выдачу/Акварель - только в `deep` (Фаза 3). Имена тулов - голые (серверы подключены глобально, резолвятся в рантайме).

---

## По фазам

### Фаза 0 - site-scanner (только режим «сайт», последовательно)

| Тул | Что даёт | Лимит |
|---|---|---|
| `seo_fetch_batch` (profile="outline") / `seo_fetch_page` | Скачать HTML страниц (текущие H1/Title/Description); веер по URL - `seo_fetch_batch(urls=[...], profile="outline")`. sitemap.xml / robots.txt (не-HTML) - `seo_fetch_page(url)`, тело придёт в `body_raw`. Деградированный fallback - `web_fetch` (теряет мету/структуру/HTTP-статус) | по числу страниц (с потолком) |
| `robots_txt` (Keyso) | Найти Sitemap-директиву если /sitemap.xml не отвечает | 0-1 |
| зашитый список кодов (Москва 213, СПб 2, ...) + дефолт 213 | Код региона если не в стандартном списке (дерево регионов Wordstat живым инструментом не отдаётся; берём код из зашитого списка / дефолт 213) | 0 |

Потолки (дефолты, настраиваемы): ≤ 300 URL из sitemap, ≤ 80 fetch страниц. При пустом `<title>` (JS-рендер) - пометить «не удалось прочитать», не пропускать тихо.

### Фаза 1 - metatag-researcher (последовательно, БАТЧ на весь проект)

| Тул | Что даёт | Вызовов | Параметры |
|---|---|---|---|
| `wk_check_frequency` | Частотность none/phrase/exact | **3** (по оператору) | `keywords` = СТРОКА через `\n` (до 1000, надёжно 200-300); `geo` = числовой код Яндекса |
| `arsenkin_commerce` | Comm (0-1) + Geo (1/0) | **1** (все маркеры/варианты) | `queries=[...], region=<код>, se=1` (se=1 = Яндекс) |
| `jm_suggest` | Подсказки Яндекса + частотность | **1** (в основном для bulk-обогащения) | `keywords=[...], region, iterations=1, with_freq=true, freq_region=<код>, alphabets=["ru"]` |
| `jm_account` | Баланс перед JM-вызовом | 0-1 | страховка |
| зашитый список кодов + дефолт 213 | Код региона (фолбэк) | 0 | только если регион не в списке; дерево регионов Wordstat живым инструментом не отдаётся - берём код из зашитого списка / дефолт 213. Геозависимость запроса при необходимости проверяет `arsenkin_commerce` |

Если маркеров > 300 - бить `wk_check_frequency` пакетами по 200. `arsenkin_commerce` и `jm_suggest` - одним массивом.

### Фаза 2 - select-variations.mjs (скрипт, без MCP)

Детерминированный отбор: отсев `Comm < 0.4`, сорт по `Exact`, топ-формы на страницу.

### Фаза 3 - metatag-writer

**deep (1 вызов = 1 страница, ПАРАЛЛЕЛЬНО):**

| Тул | Что даёт | Вызовов/стр | Параметры |
|---|---|---|---|
| `arsenkin_top` | ТОП-10 Яндекса по форме | до 5 (по топ-формам) + резерв | `queries=["форма"], se=[{"type":2,"region":<код>}], depth=10, is_snippet=true` |
| `arsenkin_commerce` | Контроль Comm финальной формы | 1 | как в Фазе 1 |
| `arsenkin_parse` (mode=meta) | Реальные `<title>`/`<description>`/H1 ТОП-10 | 1 (+ опц.1) | `queries=[10 URL], mode="meta", region=<код>, se=1` |
| `arsenkin_parse` (mode=highlights) | Жирные слова сниппетов (CTR) | 1 | `queries=["форма"], mode="highlights", region=<код>, se=1, hl_depth=10` |
| `jm_text_check` (Акварель batch) | Рекоменд./водяные слова + медиана по Title конкурентов | 1 | `mode="batch", marker="форма", titles=[...], title_domains=[...], lang="ru", search_engine="yandex"` |
| `jm_text_check` (Акварель single) | Балл нашего H1, затем Title | 2 | `mode="single", keyword="форма", text="<H1/Title>", lang="ru", search_engine="yandex"` |
| `jm_account` | Баланс перед Акварелью | 0-1 | страховка |

Итого deep: ~9-11 MCP на страницу.

**bulk (чанками, БЕЗ MCP):** генерация по PLAYBOOK + данные `research.json` (частотность/Comm/Geo/подсказки из Фазы 1). MCP не зовёт.

### Фаза 4 - build-metatags-xlsx.mjs + verify-metatags.mjs (скрипты, без MCP)

### Drive - /share-metatags

| Тул | Что даёт |
|---|---|
| `mcp__gdrive-piotr__uploadFile` | Загрузка A7 + конверсия в Google Sheet |
| `mcp__gdrive-piotr__deleteItem` | Удаление старого файла при `--redo` |

## НЕ использовать

| MCP | Почему |
|---|---|
| `jm_text_generate` / `jm_text_analyze` | Для написания статей, не метатегов |
| `jm_semantic_pack` | Это /seo-struktura; запросы уже есть в pages.json |
| `domain_keywords` / `keyword_info` (Keyso) | Маркеры уже определены в структуре/таблице (кроме редкого фолбэка в researcher, если маркер пуст) |
| Webmaster (wm_*), Метрика (ym_*) | Доступы клиента не нужны |
| SpeedyIndex | Индексацию не проверяем |

---

## Бюджет

- **deep:** Фаза 1 ~5 батч-вызовов на проект + ~9-11 на страницу в Фазе 3. Для 10 страниц ≈ 5 + 100 = ~105 MCP.
- **bulk:** ~5 батч-вызовов на ВЕСЬ проект, в Фазе 3 MCP нет. Для 100 страниц ≈ 5 MCP.

Это и есть смысл переосмысления: дорогое - один раз пакетом, дорогое-per-page - только в deep и параллельно.

## Ошибки MCP

| Ошибка | Действие |
|---|---|
| `wk_check_frequency` падает на больших пакетах | Бить по 200, при 25 в deep - по 10-15 |
| `arsenkin_commerce` частичный/таймаут | Один повтор тем же массивом; непроверенные пометить `not_verified`, продолжить |
| `arsenkin_top` таймаут на одной форме | Повторить только её; < 10 результатов - принять, % от факт. глубины |
| `arsenkin_parse meta` мусор у всех 10 | Повтор через минуту; иначе пометить + работать по PLAYBOOK без эталонов |
| `arsenkin_parse highlights` пусто | Некритично, продолжить |
| `jm_text_check` таймаут (batch) | Урезать до 10 уникальных доменов, повтор |
| JM баланс < стоимости | `jm_account`, продолжить без Акварели + пометка «Акварель не выполнена» |
| Drive `convertToGoogleFormat:true` упал | Fallback `false` + подсказка активировать Sheets API |
| Регион не поддержан | Ближайший крупный город / 213 + пометка |
