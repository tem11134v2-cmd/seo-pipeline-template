# PLAYBOOK метатегов (H1 / Title / Description под Яндекс)

> Правила генерации метатегов. Читается агентами `metatag-writer` (обе глубины) и `metatag-researcher` (оси вариаций). Источник - авторская методология услуги У4. Терминология: «маркер» = главный поисковый запрос страницы; «форма» = выбранный вид маркера (число/порядок/приставка); «Акварель» = `jm_text_check`, проверка релевантности текста запросу.

---

## 0. Главные инварианты

- **H1, Title, Description генерируются под маркер страницы** (самый частотный целевой запрос) + УТП клиента из A2/брифа.
- **Точное вхождение маркера (или выбранной формы) ОБЯЗАТЕЛЬНО** в H1 и в Title (без смены числа/падежа). В Description маркер - в первых 1-3 словах.
- **Маркер никогда не режется** при сокращении длины.
- **УТП и характеристики - только из A2/брифа.** Не выдумывать, не копировать у конкурентов.
- **Запрещённые формулировки из A2** проверяются перед финализацией каждого метатега.
- **Длинное (—) и среднее (–) тире запрещены. Только дефис (-).** (Разделитель в Title - см. ниже, но это не тире в тексте.)

## 1. H1 по типам страниц

| Тип | Правило H1 |
|---|---|
| `home` (главная) | Бренд + основное направление + регион. Напр. «ХХХ - смесители для кухни и ванной в Москве» |
| `category` (категория/каталог) | Множественное число, точное вхождение маркера, **4-6 слов** |
| `subcategory` (подкатегория) | Уточнённый маркер, множественное число |
| `service` (услуга) | Маркер + выгода/квалификатор, **до 8 слов** |
| `subservice` (подуслуга) | Уточнённый маркер + ключевая характеристика |
| `product` (товар) | Название товара + ключевая характеристика (единственное число) |
| `info` / `article` (информационная) | Формулировка запроса (вопрос/утверждение) |

**Длина H1:** категории 4-6 слов; услуги ≤8 слов; товары/статьи - без жёсткого верха; **не короче 3 слов** (кроме «Контакты», «О компании»). В режиме `deep` дополнительно: длина H1 в диапазоне ТОП-10 конкурентов, ближе к медиане.

**Топоним в H1** - только если `Geo=1` И регион ≠ Москва (московская выдача и так сужена) ИЛИ проект явно региональный.

**Запрещённые H1:** ❌ «Преимущества [маркера]» (нет реальной выгоды), ❌ «Наша компания» / «Наши услуги», ❌ «Заполните форму» / «Свяжитесь с нами».
**Правильно:** маркер + конкретная выгода/число; множественное число для категорий; язык клиента, без канцелярита.

## 2. Title (≤ 60 символов, цель 50-58; в deep - медиана ТОП-10 ±5)

**Каркас:** `[Маркер/форма, точное вхождение] [разделитель] [коммерч. приставка / УТП] [топоним если уместно] [бренд если уместно]`

**Использование данных:**
- `deep`: встраивать слова из пересечения «Акварель-рекомендованные ∩ highlights» первыми; рекомендованные Акварелью - как УТП-слова; **никогда не использовать «водяные» слова Акварели** (релевантность < 0.3).
- `bulk`: брать топ-3 подсказки (`jm_suggest`) с `Exact > 50`; если в них есть коммерческий триггер (купить/заказать/цена/стоимость/каталог/магазин), которого нет в маркере - встроить; если есть полезный квалификатор (недорого/в наличии/с доставкой/производитель/акция), совпадающий с УТП из A2 - встроить как УТП-слово. Если подсказок нет: по умолчанию «купить» для каталогов, «заказать» для услуг, «купить» для товаров.

**Разделитель:** в `deep` - доминирующий в нише («голос ниши»: чаще всего `|`, реже `-`, `:`); в `bulk` - дефис или вертикальная черта.

**Сокращение если > 60** (по порядку): (1) убрать бренд, (2) ужать УТП-слова до одного эпитета, (3) убрать модальность («лучшие»/«топ»), (4) ужать разделитель (до пробела), (5) убрать топоним **только если выдача гео-независима**, (6) **маркер не трогать никогда**. Если всё равно > 60 - вывести как есть с пометкой длины и спросить пользователя.

**Запрещённые Title:** ❌ восклицательные знаки, ❌ КАПС (кроме GPS/LED/IT), ❌ кликбейт («Шок», «Вы не поверите»), ❌ двойные пробелы, ❌ запрещённые формулировки из A2.

## 3. Description (≤ 160 символов, цель 140-158; в deep - медиана ТОП-10 ±10)

**Каркас:** `[Глагол действия / маркер первым]. [УТП-1 из A2 с числом]. [УТП-2 из A2 - сервис]. [Призыв (CTA)].`
Вариант для услуг: `[Описание услуги] [в городе]. [Срок/гарантия]. [Социальное УТП]. [CTA].`

**Правила:**
- Маркер - в первых 1-3 словах, точное вхождение.
- **2-3 УТП только из A2/брифа**, предпочтительно с числами/проверяемой конкретикой. Категоризировать УТП: технические / сервисные / социальные / ассортимент; выбрать 2-3 сильнейших.
- Description **дополняет** Title, не дублирует.
- CTA в конце (императив или ценностное предложение).
- Топоним - если выдача гео-зависима и регион ≠ Москва.
- Если УТП у клиента мало (1-2) или «водянистые» (индивидуальный подход, высокое качество) - **Description короче, без воды** (Яндекс не штрафует короткое описание, но штрафует переспам).
- `deep`: подсказки/highlights с сервисными квалификаторами (доставка/гарантия/срок/акция/производитель) встраивать как сервис-описание; для Description они вторичны - база это УТП из A2.

**Сокращение если > 160** (по порядку): убрать топоним (если уже в Title) → ужать CTA → объединить два УТП → убрать одно УТП (оставить 2 сильнейших) → убрать глагол действия (начать с маркера). Если всё равно > 160 - вывести как есть + пометка + спросить.

**Запрещённые Description:** ❌ «Наша компания предлагает широкий ассортимент», ❌ «Индивидуальный подход к каждому клиенту», ❌ «Высокое качество по доступным ценам», ❌ «Команда профессионалов с многолетним опытом», ❌ «Мы являемся лидерами рынка», ❌ КАПС / двойные пробелы, ❌ «!» в середине (один допустим в финальном CTA), ❌ запрещённые формулировки из A2.

## 4. Оси вариаций маркера (для researcher и deep-writer)

Из одного исходного маркера строится пул **15-25 вариантов** (мин 12, макс 30) по осям:
- **Число:** единственное / множественное. ВАЖНО: в Яндексе это часто **разные интенты** (кухня/кухни, окно/окна, дверь/двери, диван/диваны) - всегда тестировать обе формы.
- **Порядок слов.**
- **Коммерческая приставка:** купить / заказать / цена / стоимость / каталог / магазин.
- **Топоним.**
- **Морфология / уточнения.**
- **Доп. характеристика** (для товаров).

Дедуп только точных дублей. Не генерировать варианты с запрещёнными словами из A2.

## 5. Пороги Comm / Geo (из arsenkin_commerce)

`Comm` (0-1) = доля коммерческих сайтов в ТОП-10. `Geo` (1/0) = геозависимость выдачи.

| Тип страницы | OK | Borderline | Низкий (отсечь/синоним) |
|---|---|---|---|
| `home` | `>= 0.30` | `0.15-0.29` | `< 0.15` (главная часто mixed-intent) |
| `category`/`service`/`product`/`subcategory`/`subservice` | `>= 0.60` | `0.40-0.59` | `< 0.40` |

- В `deep` STEP-отборе: формы с `Comm < 0.4` - **жёстко отсекаются** (информационный интент), `0.4-0.7` допускаются с пометкой, `>= 0.7` - идеально.
- `Geo` отсутствует в ответе → дефолт `1` (безопасно для коммерческих ниш) + пометка.
- `info`-страницы коммерциализацию не проверяем.

## 6. Анализ выдачи (deep, агрегаторы)

Считать в ТОП-10 по форме: % сайтов нашего типа; % агрегаторов/маркетплейсов (**Авито, Яндекс.Услуги, 2ГИС, ЦИАН, Дром, Profi.ru, Юду, Ozon, Wildberries, AliExpress, Маркет, Tiu.ru, Pulscen** + домены из A3); % прямых конкурентов из A2; коммерческая структура.

Пороги агрегаторов: 0-2 - перспективно; 3-5 - тяжело; **6+ - почти непробиваемо**. Отбросить форму, где агрегаторов 6+ И сайтов нашего типа < 3. Если все формы провалились - пробовать резервные, иначе сообщить «ниша забита агрегаторами», обсудить более узкий маркер (запросы 4-10).

Парсинг меты ТОП-10 (`arsenkin_parse mode=meta`) даёт реальные `<title>`/`<meta description>`/H1 конкурентов - эталоны длины и паттернов. Если валидных < 6 из 10 - пометить «медианы могут быть неточными». `arsenkin_top` отдаёт **сниппет-Title** (Яндекс переписывает) - только для оценки выдачи, НЕ для финального Title.

## 7. Акварель (jm_text_check) - deep

- `mode="batch"` (Title конкурентов, ~15-20 шт. + `title_domains` для авто-дедупа) → рекомендованные слова (релевантность ≥ 0.5), «водяные» (< 0.3), медиана. Медиана = эталон само-проверки.
- `mode="single"` (наш H1, затем наш Title) → %, среднее, медиана + списки релевантных/нерелевантных/служебных слов. **Description Акварелью НЕ проверяем** (он под УТП+CTA, не под маркер).
- Само-проверка: наши `single`-баллы сравниваем с `batch`-медианой конкурентов. Это сравнение РАЗНЫХ режимов Акварели → пороги **мягкие (±0.10)**:

| Наш балл vs медиана | Статус | Действие |
|---|---|---|
| ≥ медиана + 0.10 | Отлично | Принять |
| медиана ±0.10 (коридор) | Норма | Принять |
| медиана −0.20 .. −0.10 | Borderline | Принять + пометка в «Заметки» |
| < медиана − 0.20 | Плохо | Переписать |

Переписывать: убрать «водяные» слова, заменить на рекомендованные; пере-проверить; **максимум 2 итерации**, иначе пометить и оставить. Если медиана недоступна - эталон ≈ **0.55**.

## 9. Само-чек перед записью строки (обе глубины)

- Точное вхождение маркера/формы в H1 и Title.
- Маркер в Description в первых 1-3 словах.
- Title ≤ 60, Description ≤ 160.
- H1 в диапазоне типа (категория 4-6 слов и т.п.); в deep - в диапазоне ТОП-10.
- Нет запрещённых формулировок из A2 нигде.
- Нет банальностей/кликбейта/КАПСа/двойных пробелов/тире.
- УТП в Description - из A2.
- H1 → Title → Description согласованы (дополняют, не дублируют).
- (deep) Акварель пройдена для H1 и Title.
