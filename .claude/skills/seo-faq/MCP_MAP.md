# MCP-карта /seo-faq

| Шаг / агент | Инструмент | Зачем | Обязательность |
|---|---|---|---|
| 2 (гейт) | `jm_account` | проверка баланса >= 5 перед анализом | обязательно |
| 4 `faq-builder` | `jm_text_analyze` | анализ пробелов текста vs топ-10 -> недостающие ключи/N-граммы (ядро) | обязательно |
| 4 `faq-builder` | `jm_stop_domains` | исключить агрегаторы из топ-10 при анализе | желательно |
| 4 `faq-builder` | `seo_fetch_page(profile="content")` / `jm_page_optimize` | если источник - живой URL без текста (основной текст страницы) | только --url |
| 4 `faq-builder` | `jm_text_check` (Акварель) | контроль переспама итогового FAQ-текста | желательно |
| 6 Drive | `mcp__gdrive-piotr__uploadFile` | FAQ_<slug>.docx -> Google Doc (ключ `texts_folder_id`) | желательно (скип если нет ключа) |

**JM - дорогой.** Кешируй результаты анализа в `faq/NNN/pages/<slug>/` и не пересчитывай. Перед `jm_text_analyze` баланс уже проверен на шаге 2 (гейт скила), но faq-builder тоже страхуется `jm_account`.

## Замечания (выявлено на реальном прогоне)
- **Регион в `jm_text_analyze` - это код ГОРОДА, не страны.** `225` (Россия) и прочие страновые коды **не принимаются** - нужен город (Москва `213`, СПб `2`, ...). Для общероссийского/мультирегионального клиента бери **`213`** как стандарт (не `region_yandex` если там 225). faq-builder это знает и фолбэчит на 213.
- **JM-кравл URL работает по ОПУБЛИКОВАННОЙ странице.** Если страница ещё прототип (URL отдаёт 301/404), `jm_text_analyze` не снимет `src`-дифф - тогда анализ идёт по тексту страницы + бенчмарку топ-10, а `coverage_before/after` помечается как экспертная оценка (`coverage_basis`), не машинная. Это норма для фазы прототипа.
- **Стоп-домены** - агрегаторы (avito, ozon, ...) из `inputs`, чтобы анализ шёл по коммерческим конкурентам, а не маркетплейсам.
- Schema.org FAQPage генерит `build-faq.mjs` (скрипт), не агент - детерминированно.
