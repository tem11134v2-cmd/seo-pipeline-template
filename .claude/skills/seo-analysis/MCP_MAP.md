# MCP-карта для /seo-analysis

> Какие MCP-инструменты использовать на каком шаге. Принцип: экономия контекста. Не вызывай всё подряд — бери только нужное для текущего этапа.

---

## Основные инструменты

### Keyso

| Тул | Что даёт | На каких шагах | Лимит |
|---|---|---|---|
| `domain_dashboard` | DR, ТОП-10/50, трафик, страниц в базе | 1 (проверка домена клиента, если есть) + 3 (по каждому из 8-15 отфильтрованных конкурентов) | 1 + 8-15 |
| `domain_competitors` | Список конкурентов по пересечению семантики | 2 (Путь A, Путь B опорный, Путь C/D через топ-3 первичных) | 1-3 |
| `keyword_info` | SERP по запросу: ТОП-50 + частотность | 2 (Путь B/C/D — сбор кандидатов по 3-7 запросам) + 4 (SERP-анализ по 3-5 коммерческим запросам) | 3-7 + 3-5 |
| `domain_pages` | Страницы домена с числом запросов в ТОП | 5 (по каждому из 3 лидеров — топ-10 страниц) | 3 |

**Параметры:**

- Во всех вызовах Keyso передавай `base="<brief.keyso_base>"`. База определяется на шаге 1 агентом `brief-structurer`.
- `include_history=true` **НЕ нужен** — для предпроектного анализа достаточно текущих метрик.

**Кириллический IDN-домен** (`ремонт-квартир-днр.рф`) передавай **в кириллице**, не в Punycode (`xn--...`). Keyso работает с кириллической формой, Punycode даст «домен не найден».

```
domain_dashboard(domain="site.ru", base="spb")
domain_competitors(domain="site.ru", base="spb")
keyword_info(keyword="запрос", base="spb")
domain_pages(domain="leader.ru", base="spb", sort="it50|desc", per_page=10)
```

### Wordstat MCP

| Тул | Что даёт | На каких шагах |
|---|---|---|
| `mcp_yandex_search` | Поиск в Яндексе | 2 (если `brief.city_not_in_keyso == true` — локальные игроки) |
| `mcp_fetch_page` | Парсинг страницы с метаданными | 2 (проверка спорных доменов на агрегатор / инфопортал / релевантность) + 5 (скан страниц топ-3 лидеров) |

```
mcp_yandex_search(query="<запрос> <город>")
mcp_fetch_page(url="https://leader.ru/services/teambuilding")
```

### Встроенные Claude

| Тул | На каких шагах |
|---|---|
| `web_fetch` | 2 (fallback если mcp_fetch_page не работает) + 5 (то же) |

## НЕ использовать в /seo-analysis

| MCP | Почему |
|---|---|
| **JustMagic** (jm_*) | Для текстов/статей, не для предпроектного анализа |
| **Webmaster** (wm_*) | Доступы клиента не запрашиваются на предпроектном этапе |
| **Метрика** (ym_*) | То же |
| **SpeedyIndex** (speedyindex_*) | Не нужно проверять индексацию для предпроектного анализа |
| **Arsenkin** (arsenkin_*) | Дублирует Keyso для этой задачи |
| **Telegram** (tg_*) | Не относится |
| **Sheets** | Все артефакты — markdown и docx, не электронные таблицы |
| **Drive** (gdrive-*) | Расшаривание не предусмотрено в скиле (можно сделать вручную если нужно) |

## Типовой порядок вызовов (всего ~25-40 на анализ)

```
--- brief-structurer (шаг 1) ---
1.  domain_dashboard(клиент)                      # только если домен есть

--- competitor-finder (шаги 2-3) ---
2.  domain_competitors(клиент)                    # Путь A
    ИЛИ keyword_info(× 3-5 запросов) + опорный    # Путь B
    ИЛИ keyword_info(× 5-7 запросов)              # Путь C/D
3.  mcp_yandex_search(× 2-3)                      # если регион не в Keyso
4.  mcp_fetch_page(× до 5)                        # проверка спорных доменов
5.  domain_dashboard(× 8-15)                      # метрики финальных кандидатов

--- serp-verdict (шаг 4) ---
6.  keyword_info(× 3-5 коммерческих запросов)     # SERP-анализ
7.  mcp_fetch_page(× до 3)                        # тип страницы top-1/2/3 в спорных случаях

--- leader-scanner (шаг 5) ---
8.  domain_pages(× 3 лидера)                      # выбор страниц для скана
9.  mcp_fetch_page(× 9-12 страниц)                # скан смыслов

--- analysis-writer (шаг 6) ---
   (без MCP — только Read/Write/Edit)

--- build-analysis-docx.mjs (шаг 7, опц.) ---
   (без MCP — Node-скрипт)
```

**Бюджет:** ~25-40 MCP-вызовов на анализ. Если упёрся в лимит — не дробить запросы дальше, идти к следующему этапу.

---

## Когда нет домена клиента

Если `brief.domain == null`:
- Пропустить `domain_dashboard` клиента на шаге 1.
- `brief-structurer` ставит `path = "C"` (если есть `client_target_queries`) или `path = "D"` (если только ниша + регион).
- `competitor-finder` идёт путём C или D: 5-7 маркерных запросов → `keyword_info` → топ-3 первичных → `domain_competitors` каждого.
- `serp-verdict` всё равно работает: коммерческие запросы есть.
- `leader-scanner` всё равно работает: топ-3 определены `competitor-finder`.

## Что делать при ошибках MCP

| Ошибка | Действие |
|---|---|
| Keyso вернул пустые данные на домен клиента | `path = "B"`, метку «нет данных Keyso» в `brief.domain_dashboard_snapshot.note`, продолжать |
| Keyso вернул пустые данные на домене конкурента | Пометить `"keyso_data": "missing"` в его записи, оставить в списке (тип и тематика важнее) |
| `mcp_fetch_page` 403/404/timeout | Попробовать `web_fetch`. Если оба не работают — `"fetch_failed": true`, пропустить страницу, продолжать |
| `mcp_yandex_search` не работает | Пропустить шаг 1.5 (расширение для регионов вне Keyso), отметить в `candidates.json.note_yandex_search` |
| Превышен бюджет MCP (>40 вызовов) | Прекратить добор кандидатов, перейти к следующему этапу с тем что есть |
