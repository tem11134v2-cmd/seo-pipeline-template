# fix-article

Точечная правка готовой статьи. Использует `article-fixer`.

## Аргументы

```
/fix-article <NNN> "<описание правки>"
```

- `NNN` — номер статьи (префикс папки в `articles/`).
- `<описание>` — что поправить.

## Алгоритм

### 1. Найти папку

`articles/NNN-*/` — взять первую подходящую. Если не найдено — сообщить пользователю и остановиться.

### 2. Определить целевой файл

По описанию правки и состоянию папки:
- Если есть `output.html` — обычно правим его.
- Если есть `tilda/head.html` или `tilda/t123.html` — спросить пользователя или применить правку к `output.html`, после чего перезапустить `tilda-split.mjs`.
- Если есть только `article.md` (статья ещё не собрана) — править его.

Если непонятно — спросить пользователя.

### 3. Делегировать `article-fixer`

Маркер: маркер на этот раз необязателен — `article-fixer` возвращает дифф, не новый файл.

Промт:
```
article_dir: <abs path>
description: <описание правки>
project_root: <...>
```

### 4. Если правили `output.html` и есть Тильда

Прочитать `ЗАКАЗЧИК.md`. Если Платформа = «Тильда» И существует `<dir>/tilda/`:
```
node .claude/scripts/tilda-split.mjs <dir>
```

Сообщить:
> «Тильда-блоки переразделены: tilda/head.html и tilda/t123.html обновлены.»

### 5. Готово

Вывести дифф (уже выведен субагентом) + указать, какие файлы изменены.

## Запреты

- Не запускай весь цикл `write-article` заново.
- Не правь `meta.json` — состояние не меняется.
- Не используй длинное тире (—) и среднее (–). Только дефис (-).
