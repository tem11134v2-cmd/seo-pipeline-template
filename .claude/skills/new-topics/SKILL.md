# new-topics

Собирает 15-25 информационных тем для блога текущего проекта.

## Аргументы

`/new-topics` — без аргументов. Опционально: `--queries "запрос1, запрос2"` для затравочных запросов.

## Алгоритм

### 0. Проверка: мы в main (без worktree)?

```bash
GIT_DIR=$(git rev-parse --git-dir)
COMMON_DIR=$(git rev-parse --git-common-dir)
```

Если разные — отказать:
> «`/new-topics` правит общий файл `topics.xlsx`. Эту команду нужно запускать в main-сессии (без галочки worktree). Открой основную папку проекта без worktree.»

### 1. Прочитать `ЗАКАЗЧИК.md`

Извлечь:
- Нишу
- Регион (код города)
- Существующий блог клиента (если указан в перелинковке)

### 2. Делегировать `topic-generator`

Маркер:
```
.claude/tmp/expected-topic-generator-<run_id>.txt:
  topics.json
```

Промт:
```
project_root: <путь>
ниша: <из ЗАКАЗЧИК.md>
регион: <из ЗАКАЗЧИК.md>
домен: <из ЗАКАЗЧИК.md>
затравочные запросы: <если переданы>
Собери 15-25 тем по PHASE-1 (4 подшага сбора + формирование списка). Запиши topics.json.
```

### 3. Пауза после мини-отчёта

Субагент уже вывел в чат мини-отчёт сбора. Спросить пользователя:
> «Мини-отчёт сбора. Подтверди (`ок`) или предложи корректировку (`добавь темы про X`, `убери тему N`).»

Ждать ответа. Если корректировки — повторно делегировать `topic-generator` или применить через `Edit topics.json`.

### 4. Конвертация в xlsx

```
node .claude/scripts/to-excel.mjs
```

### 5. Открыть в Excel / браузере

```
PowerShell: Start-Process topics.xlsx
Bash: xdg-open topics.xlsx || open topics.xlsx
Windows cross: start topics.xlsx
```

Сообщить:
> «topics.xlsx готов: N тем, M конкурентов. Дальше: `/write-article <N>` для конкретной темы.»

## Запреты

- Не дублируй темы из существующего блога клиента.
- Не используй длинное тире (—) и среднее (–). Только дефис (-).
- Не пиши статьи — это работа `/write-article`.
