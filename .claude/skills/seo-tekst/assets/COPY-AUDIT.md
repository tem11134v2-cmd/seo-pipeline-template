# COPY-AUDIT.md - чек-лист контроля продающего текста

Файл `copy-auditor`. Прогоняется по готовому page.json ПЕРЕД сборкой HTML; нарушения чинятся на месте.
Писатель этот файл не читает (его файл - VOICE.md); проверка - отдельными свежими глазами.

**Это контроль ПРОДАЮЩЕГО текста, а не борьба с ИИ-детектором** (анти-ИИ-паттерн - забота статей, не лендингов; см. ADR-022). Порядок важности при правке:
1. **Смысл и грамотность** (П.0) - текст верен, не преувеличен, грамотен. Это первое, что бракует заказчик.
2. **Бьёт ли в боль ЦА** (П.1) - блоки закрывают боли/возражения, а не «о нас».
3. **Чистота и продающая дисциплина** (П.2-П.13) - жаргон, штампы, манипуляции, лимиты, утечки.

Механическую часть (тире, жаргон+утечки, манипуляции, H1, эмодзи, лимиты слотов) проверяет `verify-copy.mjs`; смысл, боль и тон добивает агент.

## Чек-лист

```
[ ] 0. СМЫСЛ + ГРАМОТНОСТЬ: каждое утверждение верно и не преувеличено; причинно-следственные связки держатся; орфография/пунктуация/согласование чисты; нет самопротиворечий между блоками
```

---

## П.0. Смысл и грамотность (ПЕРВЫМ - это бракуют заказчики)

Заказчики возвращают тексты за «перегнутый смысл и неграмотность», а не за «ИИ-шность». Лови именно это:

- **Преувеличения и неправда.** «В 10 раз», «всегда», «никто кроме нас», «100%», превосходные степени - верны только если это факт из `facts.json`. Нет факта -> снять степень / переписать.
- **Кривые причинно-следственные связки.** «X, поэтому Y», где Y из X не следует -> переписать. Логические нестыковки (компания приписывает себе чужую роль; «не сваливаем на подрядчиков», будучи сама подрядчиком) -> исправить.
- **Самопротиворечия между блоками.** Один факт - одно значение на всей странице (сверь с facts.json).
- **Грамотность.** Рассогласование падежей/чисел, проглоченные слова, кривые обороты, пунктуация -> вычистить. Читай каждый блок как корректор.

## П.1. Бьёт ли в боль ЦА

- Каждый блок должен закрывать боль/страх/возражение (сверь с `blueprint.blocks[].covers`; при сомнении - с `audience.json` summary). Блок «про нас» без привязки к боли - переписать под боль или сигналить в сводке как слабый.
- Формулировки боли - узнаваемые, словами клиента, не маркетинговый язык.

## П.2. Утечка внутренней кухни + жаргон маркетолога -> 0

Клиент покупает результат, а не методику и не наши инструменты. Запрещено в клиентском тексте:

| Запрещено | Замена |
|---|---|
| **Сургай** и любые названия методик | убрать совсем (это наша кухня) |
| **кастдев** / customer dev | «созваниваемся с вашими клиентами» |
| JTBD | «зачем покупают» |
| CJM | «путь клиента» / убрать |
| сегмент / персонаж / аватар | «ваши клиенты» / «кто покупает» |
| УТП / USP | «чем вы отличаетесь» (фактом, не ярлыком) |
| оффер / лид / конверсия / воронка | бытовое слово или убрать |

## П.3. Регистр по брифу - без развязности

«Ты» и разговорный тон допустимы (если не противоречат брифу), но развязный сленг подрывает доверие:

| Стоп | Почему |
|---|---|
| «накосячили», «колхозно», «зоопарк», «болтун» | развязность, несолидно |
| «реально», «по-честному» | дворовая интонация |
| «нарвались» / «попали» / «кинули» | негатив-проекция на читателя |
| «без условий», «не берём таких» | высокомерие |

Тон сверять с тональностью брифа (inputs).

## П.4. Метафоры и сравнения

Максимум 1 образное сравнение на страницу, точное и уместное регистру. Лишние / кривые / снижающие смысл - убрать, заменить прямой конкретикой. «Цифра против цифры» (реальное сравнение фактов) - не метафора, разрешена.

## П.5. Штампы (мнение без факта) -> конкретика

| Запрещено | Чем заменить |
|---|---|
| «Индивидуальный подход» | механизм: «Прораб на связи. Каждые 7 часов - фотоотчёт в WhatsApp» |
| «Многолетний опыт» | «9 лет, 300+ проектов» |
| «Профессиональная команда» / «Опытные специалисты» | состав по ролям; стаж + образование |
| «Гарантируем результат» | «метрика X в договоре, превышение - компенсация» |
| «Лучшие на рынке» / «Самые низкие цены» | цифры (доля, клиенты, «экономия до 60 000 ₽ за счёт...») |
| «Полный цикл услуг» | перечень того, что входит |
| «уникальный/лучший/качественный», «широкий ассортимент», «лидеры рынка» | убрать или заменить фактом |

## П.6. Самозащита и частица «не»

«Не как у других», «не шаблонное КП», «больше не придётся...», «забудьте о том, что...» - упоминание чужого минуса рождает сомнение. Замена: утвердительный факт о себе ЛИБО «цифра против цифры» («5 дней vs 3 недели»). «Защищены от X» - ок как замена отрицания, но не как выпад в конкурента. Максимум одно «не» на блок.

## П.7. Аббревиатуры

В Hero/H1 - 0. Глубже - расшифровка при первом упоминании ИЛИ бытовое слово (SLA -> «время реакции в договоре», CRM -> «система учёта заявок»). Профжаргон ниши («сработка», «облёт», «обвес») - пояснить в скобках или заменить.

## П.8. Манипуляции (тест ФАС)

| Запрещено | Замена |
|---|---|
| Перечёркнутая цена без реальной скидки | честная сравнительная таблица |
| «Только сегодня» без дедлайна | убрать или реальная дата |
| «Осталось 3 места» с выдуманной цифрой | реальный остаток или убрать |
| **Арифметика на лету** (NPS 76% -> «90% рекомендуют») | только исходная цифра из FACTS |
| «Бонус» из того, что и так входит | не подавать как бонус |

## П.9. Цифры = FACTS + плейсхолдеры

Каждое число сверить с `facts.json`. Выведенных арифметикой нет. Чего нет -> `[ЗАПОЛНИТЬ: что именно]`. Заглушки без сленга («ща докрутим» запрещено). Реалистичные числа - только там, где формат сам читается как заглушка (прайс, телефон, email); никогда в «отзыве/кейсе/проценте успеха».

## П.10. Карта повторений

Каждый факт/УТП <= 1-2 раза на страницу. Дубли «для симметрии сетки» - вырезать. Скидка/акция - максимум в 1-2 местах (одно ценовое сообщение).

## П.11. Лимиты длины

- H1 <= 60 символов, без перечислений; метафора/числовой фильтр аудитории - не в H1.
- CTA-кнопка - глагол действия + конкретика, 15-30 символов.
- Плашка Hero = заголовок + 1 строка.
- Char-диапазоны слотов - по `limits` из blueprint, допуск ±15% (несущее ограничение вёрстки).
- `page.description` <= 160 символов.

## П.12. Формула «за счёт» + H2-выгода

`<Вы-польза в цифрах> за счёт <свойство>` - замкнута на свойство. H2 = выгода с цифрой, не ярлык секции («Почему мы?», «О компании» - мёртвые). Тест «по одним заголовкам»: H1+H2 складываются в продающую историю.

## П.13. OPSEC + типографика

Деликатные ниши (оборонка/медицина/опасные производства): личность/геолокацию/комплектующие/цифры результатов обезличить и пометить `[требует согласия]`. Типографика: длинное/среднее тире (— –) -> дефис (-); эмодзи - один источник, не на первом экране и не в H1/H2; текстовые стрелки в Hero -> SVG кита.
