# COPY-AUDIT.md - чек-лист контроля копирайта (14 пунктов + стоп-листы)

Файл `copy-auditor`. Прогоняется по готовому page.json ПЕРЕД сборкой HTML; нарушения чинятся на месте.
Писатель этот файл не читает (его файл - VOICE.md); проверка - отдельными свежими глазами.
Механическую часть пунктов 1-13 проверяет `verify-copy.mjs`; семантику и пункт 14 добивает агент.

## Чек-лист 14 пунктов

```
[ ] 1. Самозащита «защищены от / не как у / без того» -> 0
[ ] 2. Жаргон маркетолога (JTBD/CJM/УТП/оффер/сегмент) в клиентском тексте -> 0
[ ] 3. Аббревиатуры в Hero/H1 -> 0; глубже - расшифрованы
[ ] 4. Дворовая лексика, тройные отрицания -> 0
[ ] 5. Манипуляции (перечёркнутые цены / срочность / арифметика на лету) -> 0
[ ] 6. Плейсхолдеры: служебное вне продающего текста; без сленга; цены - факт из FACTS или [ЗАПОЛНИТЬ]
[ ] 7. Все цифры сверены с FACTS, нет выведенных арифметикой
[ ] 8. Карта повторений: каждый факт/УТП <= 1-2 раза
[ ] 9. Лимиты длины: шапка 1-2 слова/элемент, кнопка 1 слово, H1 <= 60 символов без перечислений
[ ] 10. Формула «за счёт» замкнута на свойство
[ ] 11. H2 - выгода, не ярлык секции
[ ] 12. Эмодзи: один источник, не в H1/H2 первого экрана
[ ] 13. OPSEC (деликатная ниша): чувствительное обезличено / [требует согласия]
[ ] 14. Маркеры ИИ-текста -> 0 (тире, слова-маркеры, симметрия блоков, штампы)
```

---

## П.1. Самозащита (спор с конкурентом, которого читатель не упоминал)

«Не как у других» / «не шаблонное КП» / «не портянки PDF» - само упоминание чужого минуса создаёт сомнение там, где его не было: «значит, такие же».

❌ «Не как у других», «Не отправляем шаблонные КП», «Не «когда сможем»», «Не SMM и не «креатив»»

**Замена:** утвердительный факт о себе ЛИБО «цифра против цифры»:
- ✅ «5 дней vs 3 недели на конструкторе»
- ✅ «Реакция от 15 минут зафиксирована в SLA... -> в договоре»
- ✅ «Старт за 5 рабочих дней - собственная платформа»

**Частица «не» (подсознание её опускает):** «не будете обмануты» -> «вы защищены от обмана»; «не подведём» -> «вы ограждены от срывов». Максимум одно отрицание на блок.

## П.2. Маркетинговый жаргон -> язык предпринимателя

Клиент покупает результат, а не методику. Запрещено в клиентском тексте: `customer dev`, `JTBD`, `CJM`, `маркетинг-анализ`, `сегмент`/`персонаж`/`аватар`, `УТП`/`USP`, `оффер`/`лид`/`конверсия`/`воронка`/`прототип`.

| Жаргон | Замена |
|---|---|
| customer dev | «созваниваемся с вашими клиентами» |
| JTBD | «зачем покупают» |
| CJM | «путь клиента» / убрать |
| маркетинг-анализ | «разбор спроса и конкурентов» |
| сегмент / персонаж / аватар | «ваши клиенты» / «кто покупает» |
| УТП / USP | «чем вы отличаетесь» (фактом, не ярлыком) |
| оффер / лид / конверсия / воронка | бытовое слово или убрать |

## П.3. Аббревиатуры

В **Hero/H1 аббревиатур НЕТ вообще (0)**. Глубже - расшифровка при первом упоминании ИЛИ бытовое слово. Даже если заказчик использовал термин в брифе.

| Аббревиатура | Замена |
|---|---|
| SLA | «время реакции, зафиксированное в договоре» |
| CRM | «система учёта заявок» |
| ICP / ЦА | «ваши клиенты» |

Профжаргон ниши («сработка», «облёт», «обвес», «прошив») - пояснить в скобках или заменить. Тест: страница понятна человеку, который слышит про нишу впервые.

## П.4. Дворовая / конфронтационная лексика

| Стоп | Правило |
|---|---|
| «реально», «по-честному» | дворовая интонация, подрывает солидность |
| «нарвались» / «попали» / «кинули» | негатив-проекция на читателя |
| «без условий», «не берём таких» | высокомерие |
| тройные отрицания («Не диспетчер. Не оператор. Не робот») | максимум одно отрицание на блок |
| конфронтация с конкурентом по имени | только «цифра против цифры», без имён |

Тон сверять с тональностью брифа.

## П.5. Манипуляции (тест ФАС)

| Запрещено | Замена |
|---|---|
| Перечёркнутая цена без реальной скидки в кассе | честная сравнительная таблица |
| «Только сегодня» без дедлайна | убрать или реальная дата |
| «Осталось 3 места» с выдуманной цифрой | реальный остаток или убрать |
| **Арифметика на лету** (NPS 76% -> «90% рекомендуют»; годовой ÷ 12) | только исходная цифра из FACTS |
| Сравнение разнородного как «скидка» | сравнивать сопоставимое |
| «Бонус» из того, что и так входит | не подавать как бонус |

Тест: перечитай блок глазами проверяющего ФАС - есть риск «введения в заблуждение»? Переписать.

## П.6. Стандарт плейсхолдеров

- Незаполненное: «Требует уточнения» или `[ЗАПОЛНИТЬ: что именно]`.
- БЕЗ сленга в заглушке («ща докрутим», «потом допилим»).
- Цена: подтверждённая из FACTS либо явный `[ЗАПОЛНИТЬ]`; «реалистичные» полузаглушки, неотличимые от факта, запрещены.
- Служебные пометки собственнику - в `fill_notes`, не в продающий текст.
- Реалистичные числа допустимы только там, где формат сам читается как заглушка: прайс (`9 990 ₽`), телефон (`+7 (XXX) XXX-XX-XX`), email (`info@company.ru`).
- **Никогда** в «отзыве», «кейсе», «достижении», «проценте успеха».

## П.7. Цифры = FACTS

Каждое число на странице сверить с `facts.json`. Выведенных арифметикой нет. Чего нет в FACTS -> `[ЗАПОЛНИТЬ]`. Молча достраивать факты запрещено.

## П.8. Карта повторений

Каждый факт/УТП <= 1-2 раза на страницу. Дубли «для симметрии сетки» - вырезать. Скидка/акция - максимум в 1-2 местах (одно ценовое сообщение).

## П.9. Лимиты длины

- H1 <= 60 символов, без перечислений; метафора и числовой фильтр аудитории - не в H1 (в подзаголовок/плашки).
- Кнопка - короткая, глагол действия.
- Плашка Hero = заголовок + 1 строка.
- Char-диапазоны блоков - по blueprint/BLOCKS.md, допуск ±15%.

## П.10. Формула «за счёт»

`<Вы-польза в цифрах> за счёт <свойство>` - замкнута на свойство, не обрывается на пользе. «Сэкономите до 30%» без «за счёт чего» - оборванная формула.

## П.11. H2 = выгода

«Почему мы?», «Наши преимущества», «О компании», «Как мы работаем» - мёртвые H2, заменить на выгоду с цифрой. Не начинаются с «Наши/О нашей/Почему мы». Тест «по одним заголовкам»: H1+H2 подряд складываются в продающую историю.

## П.12. Эмодзи

Один источник на страницу, не в H1/H2 первого экрана.

## П.13. OPSEC (деликатные ниши: оборонка, медицина, опасные производства)

| Чувствительное | По умолчанию |
|---|---|
| Личность основателя / команды | обезличить, `[требует согласия]` |
| Геолокация производства | город/регион убрать или обобщить |
| Происхождение комплектующих | не раскрывать без согласия |
| Места / даты съёмок | убрать привязку |
| Конкретные цифры результатов | `[требует согласия]` |

## П.14. Маркеры ИИ-текста (НОВЫЙ - текст не должен опознаваться как сгенерированный)

### 14а. Типографика
- Длинное тире (—) и среднее (–) -> заменить на дефис (-). Маркер ИИ номер один.

### 14б. Слова-маркеры -> 0
- Вводные: «Важно отметить», «Стоит отметить», «Следует подчеркнуть», «Как известно», «Не секрет, что», «В современном мире», «На сегодняшний день»
- Канцеляризмы: «является», «данный», «осуществлять», «в рамках», «в целях», «посредством», «вышеуказанный»
- Шаблонные переходы: «Давайте рассмотрим», «Таким образом», «Подводя итог»
- Избыточные усилители: «очень важно», «крайне необходимо», «значительно улучшает», «существенно повышает» -> цифра вместо усилителя

### 14в. Штампы (мнение без факта) -> конкретика

| Запрещено | Чем заменить |
|---|---|
| «Индивидуальный подход» | конкретный механизм: «Прораб на связи. Каждые 7 часов - фотоотчёт в WhatsApp» |
| «Многолетний опыт» | «9 лет, 300+ проектов» |
| «Профессиональная команда» / «Опытные специалисты» | состав по ролям; стаж + образование + сертификаты |
| «Гарантируем результат» | «метрика X в договоре, превышение - компенсация» |
| «Лучшие на рынке» / «Самые низкие цены» | цифры (доля, клиенты, «экономия до 60 000 ₽ за счёт...») |
| «Полный цикл услуг» | перечень того, что входит |
| «Качественный сервис» / «Современные технологии» | конкретная характеристика / список технологий |
| «уникальный/лучший/качественный» | убрать или заменить фактом |
| «широкий ассортимент», «лидеры рынка», «высокое качество по доступным ценам» | конкретика или вырезать |

### 14г. Симметрия блоков (структурный маркер)
- Все карточки сетки одной длины с точностью до знака -> расшатать (одна у нижней границы диапазона, одна у верхней).
- Все карточки начинаются с одинаковой конструкции («Мы делаем X, чтобы Y» × 6) -> переписать зачины: глагол / цифра / существительное / рубленое утверждение.
- Параллельные триады в каждом блоке («быстро, надёжно, выгодно») -> оставить максимум одну на страницу.
- Subhead пересказывает H2 другими словами -> заменить на раскрытие/дополнение.
- Внутри описаний предложения одной длины -> чередовать длинное/короткое.

### 14д. Тон
- Фразы безупречно гладкие, как пресс-релиз -> переписать голосом владельца бизнеса (живая конкретика, допустима честная оговорка).
- Прочитай блок вслух: спотыкаешься или «звучит как нейросеть» - переписать, не подкручивать.
