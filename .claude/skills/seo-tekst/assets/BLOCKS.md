# BLOCKS.md - библиотека из 45 конверсионных блоков для коммерческих страниц

## Что это и как читать

Библиотека контентных блоков для коммерческих страниц (Услуга, Главная, Подуслуга, категория и карточка интернет-магазина, О компании, Контакты, Доставка/оплата, Отзывы). Копирайтинг-субагент выбирает нужные блоки под конкретную страницу, адаптирует под нишу клиента и пишет текст блок за блоком.

**Как выбирается блок:**
1. **Применимость по типу страницы** - таблица применимости (раздел 4): `✅` почти всегда, `○` опционально, `-` не применимо.
2. **Покрытие** - блоки идут в сценарном порядке (Hero -> контент -> доказательства -> CTA), без дублирования смыслов (см. anti-duplication правила в разделе 3).
3. **Форма (kit shape)** - каждый блок отображается на одну из 15 форм фрагментов HTML-кита: `hero / trust-numbers / cards / steps / compare-table / pricing / accordion / portfolio / reviews / geo / tag-tiles / cta-mid / form / product-card`. Маппинг блок -> форма см. `fragments-manifest.json` (`block_to_fragment`). Непокрытые типы падают в `cards`.

**Объёмы - несущее ограничение.** Диапазоны символов на элемент, диапазоны количества элементов и общий бюджет блока (`ОБЪЁМ`) - это load-bearing constraint для генерации HTML. Текст, выходящий за диапазон, ломает вёрстку фрагмента. Копирайтер обязан попадать в диапазоны; ассемблер прототипа рассчитывает на них.

**Что НЕ пишется здесь.** Блоки **33-39** (FAQ, снятие возражений, мифы, как выбрать, гарантии, плитка тегов, перелинковка) - это SEO-слой (фаза У6-Ф2). Они **отложены в скил `/seo-faq`** и здесь НЕ пишутся. В каталоге ниже они присутствуют для полноты метрик и таблицы применимости, но помечены `[ОТЛОЖЕН -> /seo-faq]`. Исключение по источнику: отдельная страница FAQ пишется по конверсии (из анализа ЦА), а не по SEO - но в конвейере seo-tekst это всё равно зона seo-faq.

Дефисы (-) вместо тире везде по правилу проекта.

## 2. Анатомия блока

Каждый блок несёт фиксированную схему. Поля:

| Поле | Что задаёт |
|------|-----------|
| **НАЗНАЧЕНИЕ** | Конверсионная роль блока - зачем он на странице |
| **СТРУКТУРА** | Перечень элементов с диапазоном символов на элемент + диапазон количества повторяемых элементов (карточек/пунктов/строк) |
| **ОБЪЁМ** | Бюджет символов на весь блок целиком (несущее ограничение вёрстки) |
| **ФОРМАТ** | Способ подачи (карточки / таблица / аккордеон / слайдер и т.п.) |
| **ПРАВИЛА** | Жёсткие правила содержания, в т.ч. anti-duplication и пороговые правила |

### Один достоверный worked example (Блок 1: Hero)

```
НАЗНАЧЕНИЕ: Моментально доносит суть предложения, ключевые выгоды и подталкивает к действию.

СТРУКТУРА:
- Заголовок H1: 20-80 символов
- Подзаголовок: 90-150 символов ИЛИ 3-4 буллета по 40-100 символов (первая часть может быть жирной)
- CTA-кнопка: 15-30 символов
- Карточки преимуществ (3-4 шт.): заголовок 10-30 символов + описание 30-90 (компактный) ИЛИ 80-150 (расширенный)

ОПЦИОНАЛЬНЫЕ ЭЛЕМЕНТЫ (выбрать 0-1):
- Форма заявки: заголовок формы 20-50 + 2-4 поля + кнопка 15-30 + подпись под кнопкой 30-60 - вместо CTA-кнопки или рядом
- Калькулятор/квиз: заголовок 20-60 + инструкция 40-80 - вместо CTA-кнопки
- Слайдер офферов (для главной): 3-5 слайдов, каждый: заголовок 20-60 + описание 60-120 + CTA 15-30
- Видео-фон: ТЗ на видео 350-1050 (что снимать)

ОБЪЁМ: 550-1400 символов (один экран) | 1050-2450 символов (слайдер)

ФОРМАТ: Текст + карточки + (форма / калькулятор / слайдер / видео)

ПРАВИЛА:
- H1 содержит основной коммерческий запрос + выгоду или цифру
- Подзаголовок раскрывает H1, не дублирует
- CTA - глагол действия + конкретика: «Рассчитать стоимость», не «Отправить»
- Карточки - выгоды клиента, не характеристики компании
- Форма - не более 4 полей
- Подпись под формой: «Перезвоним через 15 минут» / «Бесплатная консультация»
```

**Уточнения формы Hero (из кита, shared/BLOCKS.md v3 + fragments-manifest):** ровно **1 кнопка, не 2**; SVG-стрелка-загогулина к кнопке (не текстовый `←`/`↓`); ровно **3 плашки** формата «заголовок + 1 строка объяснения» (НЕ «большая цифра + подпись»); бонус под кнопкой центрируется под кнопкой (inline-block); H1 = 44px (укладка в 3-4 строки); подзаголовок - утверждение про продукт/компанию, не призыв; без дисклеймеров в Hero; цифры-якоря (trust-numbers) НЕ дублировать, если плашки в Hero уже есть; якорь CTA -> `#lead`.

### Сводка anti-duplication и пороговых правил (verbatim)

- **Цифры != Преимущества** (блок 2 vs 17): цифры = количественные, преимущества = качественные. Не дублировать.
- **Результат != Преимущества** (блок 13): результат = то, что остаётся у клиента, не выгоды компании. Не путать.
- **FAQ нужна разметка Schema.org FAQPage** (блок 33), обязательна.
- **Не ставить возражения рядом с FAQ** (блок 34): объединить как последние 2-3 вопроса FAQ либо использовать вместо FAQ.
- **Тарифы: рекомендованный якорь mid-price** (блок 20): рекомендованный пакет = средний по цене.
- **Промежуточный CTA echo** (блок 40): текст связан с предшествующим блоком (после цен -> «Рассчитать стоимость», после кейсов -> «Хотите такой же?»).
- **Цифры-якоря НЕ дублировать с плашками Hero** (блок 1/2).
- Пороговые правила (threshold): логотипов **> 8 -> карусель** (блок 3); вопросов FAQ **> 8 -> отдельная страница** (блок 33); городов **15+ -> отдельные лендинги** (блок 32, при > 10 - группировать); характеристик товара **> 15 -> группировать** (блок 44); прайс-лист **> 8 позиций -> группировать по категориям** (блок 20); подуслуг **> 8 -> 6-8 основных + «Все услуги»** (блок 6); тегов **> 12 -> группировать/«Показать все»** (блок 38).

## 5. Глобальные правила заголовков H2/H3

Применяются ко ВСЕМ блокам, не дублируются внутри шаблонов.

- **Выделять пользу, использовать глаголы действия.**
- «Преимущества радиаторов» -> «Высокая теплоотдача, надёжность и простой монтаж»
- «Наши услуги» -> «Ремонт за 45 дней с гарантией 3 года»
- «Этапы работы» -> «От заявки до результата за 5 шагов»
- **Тест заголовков:** человек, читающий ТОЛЬКО заголовки H2, понимает суть и преимущества страницы.
- **Длина H2: 20-70 символов** | **H3: 15-50 символов.**
- **Не начинать** с «Наши», «О нашей», «Почему мы» (мёртвые H2).

Опорный сценарий каждой страницы: **захватить (Hero) -> объяснить ценность (описание/преимущества/процесс) -> доказать (цифры/кейсы/отзывы/гарантии) -> конвертировать (CTA промежуточный + одна финальная форма)**.
