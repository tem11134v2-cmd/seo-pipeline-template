# sync-from-template

Накатывает свежую машинерию из локального шаблона на текущий клиентский проект. Решает проблему «баг починен в шаблоне, но живёт в старых клонах» (улучшение #4, вариант б).

Реализация намеренно простая - тривиальный glob-copy пяти папок-машинерии + preview. **Если на практике окажется, что синк требует слияния клиентских правок внутри машинерии (клиент что-то кастомил в скрипте/агенте) - СТОП, не затирать**, оставить ручной режим (см. шаг 4). Это триггер на пересмотр подхода.

## Что синхронизируется

Единая машинерия, идентичная у всех клиентов (в worktree она read-only, никогда не кастомится под клиента):
- `.claude/scripts/`
- `.claude/agents/`
- `.claude/skills/`
- `.claude/hooks/`
- `.claude/git-hooks/`

## Что НЕ трогается (клиентское / рабочее)

- `ЗАКАЗЧИК.md`, `template.html`, `topics.xlsx`
- `articles/`, `strategies/`, `analyses/`, `structures/`, `topics/`
- `.claude/tmp/`, `.claude/handoff-requests/`
- `.claude/CLAUDE.md` - по умолчанию НЕ синкать (может содержать клиентские пометки). Если в шаблоне он менялся - показать дифф отдельно и спросить.
- `package.json`, `node_modules/`, `.git/`

## Аргументы

```
/sync-from-template [<путь к шаблону>] [--apply]
```

- `<путь к шаблону>` - абсолютный путь к локальному клону `seo-pipeline-template`. По умолчанию `~/seo-projects/template-project`. Если папки или её `.claude/scripts/` нет - стоп с просьбой указать путь явно.
- `--apply` - применить изменения. Без флага - **dry-run** (только preview, ничего не меняем). Базовый режим - dry-run.

## Предусловия

- Запуск в **main** (не worktree): синк меняет общие файлы, в worktree их заблокирует pre-commit hook.
- Рабочее дерево по машинерии чистое - иначе синк затрёт несохранённые правки.

## Алгоритм

### 0. Проверка: мы в main?

```bash
GIT_DIR=$(git rev-parse --git-dir)
COMMON_DIR=$(git rev-parse --git-common-dir)
```
Если разные - отказать: «/sync-from-template работает только в main (меняет общие файлы). Открой основную папку проекта без галочки worktree.»

### 1. Найти шаблон и проверить чистоту

1. Резолвить `<template>` (аргумент или дефолт `~/seo-projects/template-project`). Проверить, что `<template>/.claude/scripts/` существует. Иначе стоп.
2. Защита от самосинка: если `<template>` == корень текущего проекта - стоп «Это и есть шаблон, синкать нечего».
3. `git status --porcelain .claude/scripts .claude/agents .claude/skills .claude/hooks .claude/git-hooks` - если есть незакоммиченные изменения в машинерии, предупредить: «В машинерии есть несохранённые правки, синк их затрёт. Закоммить/откати сначала или подтверди.» В dry-run просто отметить.

### 2. Dry-run preview (всегда)

Для каждой из 5 папок сравнить шаблон vs текущий проект:
```bash
git diff --no-index --stat <project>/.claude/<dir> <template>/.claude/<dir>
```
(или ручное файловое сравнение). Сгруппировать по статусу:
- **добавятся** - есть в шаблоне, нет в проекте;
- **изменятся** - отличаются по содержимому;
- **удалятся** - есть в проекте, нет в шаблоне (например, удалённый из шаблона скрипт).

Вывести сводку:
```
═══ SYNC-FROM-TEMPLATE (dry-run) ═══
Шаблон: <template> @ <git -C template rev-parse --short HEAD>

scripts/   +2 новых, ~3 изменятся, -0
agents/    +0, ~1 изменится, -0
skills/    +1 новый, ~4 изменятся, -0
hooks/     без изменений
git-hooks/ без изменений

Запусти с --apply, чтобы применить.
═══════════════════════════════════
```
Если изменений нет - «Машинерия уже актуальна относительно шаблона.» Стоп.

### 3. Применение (только при --apply)

1. Для каждой из 5 папок - скопировать содержимое из `<template>/.claude/<dir>/` в `<project>/.claude/<dir>/` (перезапись существующих, добавление новых, удаление файлов которых уже нет в шаблоне - чтобы машинерия точно совпала).
2. **Не трогать** ничего вне 5 папок.
3. Закоммитить:
   ```
   git add .claude/scripts .claude/agents .claude/skills .claude/hooks .claude/git-hooks
   git commit -m "sync-from-template: машинерия обновлена из <template> @ <hash>"
   ```
   В main pre-commit hook не блокирует.
4. Если в шаблоне менялся `.claude/CLAUDE.md` - показать дифф отдельно и спросить, накатывать ли (он может содержать клиентские пометки).

### 4. Стоп-условие (триггер на пересмотр подхода)

Если при сравнении видно, что клиент **намеренно кастомил** файл машинерии (а не просто отстал от шаблона) - например, в diff видны клиентские правки, которые синк затрёт безвозвратно - **остановиться** и сообщить: «В <файл> есть клиентские правки, синк их затрёт. Реши вручную: либо вынеси правку в шаблон, либо пропусти этот файл.» Не превращать простой glob-copy в merge-движок - если задача переросла в это, остаёмся на ручном накате.

### 5. Вывод

```
═══ SYNC-FROM-TEMPLATE DONE ═══
Шаблон: <template> @ <hash>
Обновлено: scripts ~3 +2, skills ~4 +1, agents ~1
Коммит: <hash>
Совет: прогони тесты `.claude\tests\*\run.mjs` и одну статью end-to-end.
═══════════════════════════════
```

## Запреты

- Не синкать клиентские файлы (`ЗАКАЗЧИК.md`, `template.html`, `topics.xlsx`, рабочие папки).
- Не запускать в worktree.
- Не затирать локальные правки без `--apply` и показанного preview.
- Если синк требует merge (клиент кастомил машинерию) - не затирать молча, остановиться (шаг 4).
- Не использовать длинное тире (—) и среднее (–). Только дефис (-).
