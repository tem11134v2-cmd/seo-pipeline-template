# Миграции машинерии

Папка с **миграциями данных/состояния** клиентского проекта. Синк (`sync-from-template.mjs`)
копирует машинерию (файлы скриптов/агентов/скилов), но иногда обновление требует ещё и
привести в порядок **данные** проекта - то, что обычным копированием файла не починишь
(например, выкинуть из git-индекса файл, ставший производным кешем).

## Когда нужна миграция

Файл-копирование решает «у клиента старая версия скрипта». Миграция решает «у клиента
данные в формате, несовместимом с новой машинерией». Примеры:
- файл, который раньше коммитился, стал gitignored-кешем -> надо `git rm --cached` (миграция 001);
- переименование поля в meta.json по всем задачам;
- перенос/переименование рабочей папки.

Если задачу решает простое копирование файла машинерии - миграция НЕ нужна.

## Контракт

Имя файла: `NNN-kebab-name.mjs` (NNN - три цифры, определяют порядок). Каждая миграция:

```js
export const id = "001-index-json-cache";          // = имя файла без .mjs
export const description = "короткое описание что делает";

// Идемпотентна: повторный запуск на уже мигрированном проекте безопасен (no-op).
// ctx = { targetRoot, git(args)->stdout|null, log(msg) }
export default async function migrate(ctx) {
  // ... привести проект к новому формату ...
}
```

- **Идемпотентность обязательна.** Миграция сама проверяет, нужно ли что-то делать.
  Журнал применённых (`.claude/.migrations-applied.json`) - оптимизация и аудит, а не
  единственная защита: миграция должна быть безопасна и без журнала.
- `ctx.git(args)` - вызов git в корне проекта (массив аргументов), возвращает stdout
  или `null` при ошибке (не бросает).
- `ctx.log(msg)` - строка в отчёт (видна в не-`--json` режиме).
- Бросок исключения останавливает цепочку миграций (последующие не применяются),
  ошибка попадает в warnings отчёта.

## Как применяются

Движок синка после копирования файлов (и до коммита) берёт миграции, которых нет в
`.claude/.migrations-applied.json`, по порядку имён прогоняет и дописывает журнал.
Изменения миграций попадают в тот же sync-коммит. В dry-run миграции только показываются
(не применяются). Отключить: `--no-migrations`.
