# serp-verdict

Твоя задача — посмотреть на ТОП по 3-5 ключевым запросам ниши, оценить **совместимость типа сайта клиента с тем что в выдаче**, и сформулировать вердикт. Дополнительно собрать промежуточный стоп-лист (агрегаторы и нерелевантные домены из SERP) и смежные направления (Фишка 9).

## Вход

- `analysis_dir` — путь к `analyses/NNN-slug/`
- `project_root` — путь к корню проекта

## Обязательное чтение

1. `<analysis_dir>/brief.json` — `niche`, `region`, `business_type`, `assortment`, `client_target_queries`, `keyso_base`, `client_type` (если есть)
2. `<analysis_dir>/competitors.json` — финальные 6-10 + топ-3 + типы + `client_type`
3. `<analysis_dir>/candidates.json` — раздел `excluded` (агрегаторы для пополнения стоп-листа)
4. `<project_root>/.claude/skills/seo-analiz/MCP_MAP.md` — карта MCP

## Что делать

### 1. Подобрать 3-5 запросов для SERP-анализа

Возьми **основные коммерческие запросы ниши**:
- Из `brief.client_target_queries` (приоритет).
- Если их нет или мало — сгенерируй сам по схеме «<основная услуга/товар> <город>», «<подвид> <город>», «<услуга> цена», «<услуга> заказать».

Должно получиться 3-5 запросов с явным коммерческим интентом.

### 2. SERP-анализ

По каждому запросу:

```
keyword_info(keyword="<запрос>", base="<brief.keyso_base>")
```

**Кириллический IDN-домен** в SERP — оставляй в кириллице, не Punycode.

Из ответа собери ТОП-10. Для каждого домена ТОП-10 определи:

- **Тип страницы** — оцени по URL/контексту (или fetch'ом главной — точечно для топ-1/2/3):
  - `catalog` — каталог/листинг с фильтрами (для ИМ-ниши)
  - `service` — страница описания одной услуги
  - `product` — карточка товара
  - `article` — статья / обзор
  - `home` — главная
  - `landing` — лендинг (1-5 страниц на домене)
  - `mixed` — смешанный (если в ТОП-10 разные типы)

- **Интент**: `commercial` / `informational` / `navigational`

- **Агрегатор?** `true/false` (по списку из шага фильтрации `competitor-finder`)

### 3. Свести по запросу

Для каждого запроса:

```json
{
  "query": "тимбилдинг спб",
  "top10": ["a.ru", "b.ru", ...],
  "page_types_in_top10": {"catalog": 6, "service": 3, "landing": 1},
  "dominant_page_type": "catalog",
  "aggregators_in_top3": 0,
  "aggregators_in_top10": 1,
  "intent": "commercial"
}
```

### 4. Сводка по всем запросам

- **Преобладающий тип страниц** в ТОП-10 (по сумме всех запросов): `<тип>`.
- **Агрегаторы в ТОП-3**: всего раз `<N>` из `<всего>` запросов.
- **Преобладающий интент**: `<commercial/informational/смешанный>`.
- **Тип сайта клиента** (из `competitors.client_type`): `<тип>`.

### 5. Вердикт совместимости

На основе сводки определи **один из четырёх вердиктов**:

| Вердикт | Условие | Рекомендация |
|---|---|---|
| **ИДЁМ** | Преобладающий тип SERP совпадает с типом клиента, агрегаторов в ТОП-3 в сумме ≤ 1 | Продолжаем без структурных изменений |
| **КОРРЕКТИРУЕМ ТИП САЙТА** | В ТОП доминируют каталоги/листинги, у клиента нет каталога (`landing` или `small`) | Добавить каталожную структуру (категории, фильтры, карточки) |
| **МЕНЯЕМ СТРАТЕГИЮ — инфоконтент** | В ТОП доминируют статьи/обзоры (`article` > 50% позиций ТОП-10) | Основной трафик через статьи, коммерческие — посадочные |
| **ИДЁМ С ОГОВОРКАМИ** | Агрегаторы доминируют в ТОП-3 (≥ 2 из 3 в большинстве запросов) | Конкурировать за позиции 4-10, параллельно работать на самих агрегаторах |

Если сразу несколько условий — выбери приоритетный (агрегаторы > инфоконтент > коррекция типа > идём).

### 6. Промежуточный стоп-лист

Собери из:
- `candidates.json.excluded` (все исключённые при фильтрации с их причинами).
- Агрегаторы, замеченные в ТОП по запросам шага 2, которых ещё нет в `excluded` (с причиной «агрегатор в SERP»).
- Нерелевантные домены из SERP, если бросаются в глаза (с причиной «нерелевантный ассортимент по SERP-наблюдению»).

Формат:
```json
{"domain": "avito.ru", "reason": "агрегатор"}
```

(Финальный A3 будет содержать только домены без причин — это делает `analysis-writer`. Здесь — промежуточный формат с причинами для прозрачности.)

### 7. Смежные направления (Фишка 9)

Просмотри `competitors.direct` (6-10 финальных) и `brief.assortment`. Найди направления, которые:

- Встречаются у ≥3 из 6-10 конкурентов.
- **Отсутствуют** у клиента (не в `brief.assortment`).

Для каждого зафиксируй:

```json
{
  "direction": "Ортодонтия",
  "competitor_count": 4,
  "competitor_examples": ["stoma1.ru", "stoma2.ru", "stoma3.ru"],
  "recommendation": "Рассмотреть добавление"
}
```

Если смежных направлений нет — пустой массив + флаг `note`: «Ассортимент клиента покрывает основные направления конкурентов».

## Сводка в чат (5-7 строк)

- Запросы проанализированы: `<N>` (список через запятую)
- Тип SERP: преобладает `<тип>`, агрегаторов в ТОП-3 `<N>` из `<всего>`, интент `<тип>`
- **Вердикт**: `<ТИП ВЕРДИКТА>` + 1 предложение обоснования
- Стоп-лист (промежуточный): `<N>` доменов
- Смежные направления: `<N>` найдено (или «значимых пробелов не обнаружено»)
- ⚠️ Не проверено: ТОП-11-50 (только ТОП-10), сезонные колебания SERP, локальные коммерческие факторы (Карты, ЯБМ)

Если вердикт `КОРРЕКТИРУЕМ`, `МЕНЯЕМ` или `ИДЁМ С ОГОВОРКАМИ` — в конце добавь:
> ⚠️ Вердикт `<тип>` — рекомендую обсудить с клиентом до продолжения работ.
> Главное в формулировке для пользователя: что именно нужно решить ДО продолжения (одно предложение).

## Запреты

- НЕ ищи новых конкурентов — список финальных уже зафиксирован в `competitors.json`.
- НЕ делай глубокий скан смыслов страниц — это `leader-scanner` на следующем шаге.
- НЕ редактируй `brief.json`, `competitors.json`, `candidates.json` — все read-only.
- Длинное тире (—) и среднее (–) не использовать. Только дефис (-).
- Бюджет: 3-5 `keyword_info` + по необходимости 3-5 `seo_fetch_page(url, profile="outline")` для определения типа страницы топ-1/2/3 спорных случаев. ≤10 MCP-вызовов суммарно.
