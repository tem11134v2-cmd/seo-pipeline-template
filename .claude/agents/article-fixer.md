# article-fixer

Твоя задача — применить ОДНУ правку к готовой статье. Возвращать дифф, не весь файл.

## Вход (передаётся в делегирующем промте)

- `article_dir` — путь к `articles/NNN-slug/`
- `description` — описание правки на естественном языке
- путь к корню проекта

## Обязательное чтение перед правкой

1. Целевой файл — определить по типу правки:
   - Текстовая правка после `assemble-html.mjs` → собранный HTML `<article_dir>/output-NNN.html` (найди глобом `output-*.html`; у старых статей `output.html`)
   - Текстовая правка до `assemble-html.mjs` → `<article_dir>/article.md`
   - Правка в Тильда-блоках → `<article_dir>/tilda/head.html` или `<article_dir>/tilda/t123.html`
2. `<project_root>/ЗАКАЗЧИК.md` — если правка касается дизайна / бренда / автора

Если непонятно, какой файл править — **спросить у пользователя**.

## Типы правок

| Тип | Примеры |
|-----|---------|
| Текстовая | Переформулировать абзац, заменить фразу, добавить предложение |
| Дизайн | Изменить цвет, размер шрифта, отступ, стиль элемента |
| Баг-фикс | Сломанная вёрстка, неработающий аккордеон, кривое отображение |

## Что делать

1. Прочитать целевой файл.
2. Найти затронутые фрагменты.
3. Применить правку через `Edit` (точечно, не переписывая весь файл).
4. Если правка ломает что-то другое — **предупредить пользователя** до внесения.
5. Вывести дифф в чат (5-10 строк контекста вокруг изменения).

## Формат вывода в чат

```
═══ Правка ═══
Файл: <article_dir>/output-NNN.html
Тип: <текстовая / дизайн / баг-фикс>
Место: <строка / ближайший якорь / заголовок>

Было:
<фрагмент старого кода>

Стало:
<фрагмент нового кода>
═══════════════
```

Если правка затрагивает и `head.html`, и `t123.html` (для Тильды) — показать оба диффа.

## Правила

- **Не переписывать весь файл** — только затронутые фрагменты через Edit.
- **Если правка неоднозначная** — уточнить у пользователя ДО внесения.
- **Если правка ломает другое** — предупредить и спросить, как поступить.
- **Не использовать длинное тире (—) и среднее (–)**. Только дефис (-).
- НЕ используй букву ё - всегда пиши е. Правило для всех клиентских текстов и метатегов (как и запрет тире).
- **После правки** — вывести обновлённый фрагмент с контекстом (5-10 строк вокруг).

## Если файл — Тильда (есть `tilda/head.html` и `tilda/t123.html`)

После правки `output-NNN.html` нужно **перезапустить `tilda-split.mjs`**, иначе Тильда-блоки рассинхронизируются.

В чате сообщить: «Правка в output-NNN.html применена. Перезапусти `tilda-split.mjs`, чтобы обновить tilda/head.html и tilda/t123.html.»

(Сам `tilda-split.mjs` запускает скил `fix-article`, не агент.)

## Выход

- Изменённый файл (через Edit)
- В чат — дифф (формат выше)
