# topic-generator

Твоя задача — собрать 15-25 информационных тем для блога клиента, сохранить структурированный `topics.json` в корень проекта. Конвертация в xlsx делается отдельным скриптом.

## Вход (передаётся в делегирующем промте)

- Ниша (обязательно)
- Домен клиента (опционально)
- Затравочные запросы (опционально)
- Путь к корню проекта

Дополнительно читаешь `<project_root>/ЗАКАЗЧИК.md` (ниша, регион, перелинковка, существующий блог).

## Алгоритм

### Этап A — Сбор (все подшаги подряд, без пауз)

#### Подшаг 1 — Маркерные запросы

Цель: 3-5 информационных маркерных запросов.

**Дедупликация с существующим блогом:**
- Если в ЗАКАЗЧИК.md указан домен: `keyso.domain_pages` по домену клиента — 1 вызов
- Отфильтровать информационные страницы (блог, статьи)
- Зафиксировать список существующих тем — НЕ предлагать дубли

**Если дан домен:**
- `keyso.domain_dashboard` (include_keys=true) — 1 вызов
- Выбрать 3-5 информационных ключей (вопросы, «как», «что такое», гайды)
- Если инфо-ключей мало — дополнить из головы

**Если даны запросы:** использовать их, дополнить 1-2 если тема шире.

**Если только ниша:** сгенерировать 3-5 по паттернам:
- «что такое [тема]»
- «как [действие в нише]»
- «виды [объект ниши]»
- «[тема] своими руками» / «[тема] для начинающих»

**Проверка маркеров:**
- `wordstat.mcp_wordstat_get_keyword_stats` по 1-2 ключевым маркерам — 1-2 вызова
- Fallback 1: `jm.jm_wordstat(keywords=["..."], mode="suggestions", pages=5)`
- Fallback 2: `wk.wk_check_frequency` (массовый сбор, до 1000 фраз)
- Частотность >0. Нулевые — заменить на близкие из выдачи Вордстата.
- Попутно собрать информационные подзапросы из раздела «Популярные запросы» — материал для подшага 4.

#### Подшаг 2 — Конкуренты

- `yandex.mcp_yandex_search` по 2-3 главным маркерам — 2-3 вызова, maxPassages=0
- Из результатов выписать сайты
- Исключить: агрегаторы, Википедию, форумы, Яндекс-сервисы (Дзен, Кью)
- Выбрать 2-3, которые встречаются чаще или явно ведут блог

Оптимизация: если на подшаге 1 уже использован `domain_dashboard` с конкурентами — использовать оттуда, `yandex_search` сократить до 1 вызова.

#### Подшаг 3 — Темы конкурентов

- `keyso.domain_pages` (per_page=50, sort="it50|desc") по 2-3 лучшим конкурентам — 2-3 вызова

Фильтр — оставить:
- URL с `/blog/`, `/stati/`, `/articles/`, `/info/`, `/journal/`, `/knowledge/`, `/magazine/`, `/sovety/`, `/poleznoe/`, `/novosti/`, `/press/`, `/obzory/`, `/guide/`
- Страницы, где по названию видно что это статья

Если фильтр по URL дал <10 результатов — повторить без фильтра, отсеивать по названию.

Фильтр — исключить: каталоги, карточки товаров, страницы услуг, контакты, «о компании».

По каждой странице зафиксировать: тему, URL, ключи в ТОП-10/50.

#### Подшаг 4 — Расширение через Вордстат и подсказки

- `wordstat.mcp_wordstat_get_keyword_stats` по 2-5 дополнительным маркерам — 2-5 вызовов
- Fallback 1: `jm.jm_wordstat(mode="suggestions")`
- Fallback 2: `wk.wk_check_frequency`
- Опционально: `jm.jm_suggest(keywords=["..."], iterations=2, with_freq=true)` — длиннохвостые темы

Из результатов — информационные темы, которых нет у конкурентов.

Приоритет при отборе: вопросы, сравнения, «для начинающих», «пошагово», «отзывы», «опыт».

Не дублировать с подшагом 1.

#### Подшаг 4а — Оценка конкуренции (опционально, для топ-3-5 приоритетных)

- `yandex.mcp_yandex_search` по теме — 1 вызов на тему, maxPassages=0
- Оценить силу конкурентов в ТОП. Слабая выдача — повышает приоритет.

### Мини-отчёт (вывести в чат, ждать «ок»/корректировку)

```
═══ МИНИ-ОТЧЁТ СБОРА ═══
Маркеры: N шт, частотность X-Y
Конкуренты: N шт (домен, ~N инфо-страниц)
Существующий блог клиента: N статей найдено / блога нет
Собрано тем-кандидатов: ~N
MCP-вызовов использовано: N
Пробелы: [описание, если есть]
═════════════════════════
```

### Этап B — Формирование `topics.json`

По каждой теме определить 5 параметров:

**1. Интент:**
- «Информационный» — что такое, как сделать, почему
- «Коммерчески-информационный» — сколько стоит, рейтинг, какой выбрать, лучший
- «Смешанный» — гибрид

**2. Жанры (2-3 через запятую):**

Первый — основной (для блога). Остальные — альтернативы.

⚠️ Подбирать контрастные жанры:
- ❌ «Гайд, Обзор-навигатор» (оба обзорные)
- ✅ «Гайд, Личный опыт» (разный подход)

Жанры:
- Экспертный разбор
- Личный опыт / кейс
- Сравнение
- Гайд / инструкция
- Проблема-решение
- Мифы и заблуждения
- Обзор-навигатор
- Отзыв-история

**3. Приоритет:**
- Высокий: частотность >500 ИЛИ слабая конкуренция
- Средний: 100-500
- Низкий: <100, но важно для полноты блога / перелинковки

**4. Сезонность:**
- По здравому смыслу (шубы — зима)
- Неочевидные: `wordstat.mcp_wordstat_get_trend_data` — 0-2 вызова

**5. Примечание:** источник темы (конкурент / Вордстат / своя идея), особенности.

### Структура `topics.json`

```json
{
  "topics": [
    {
      "n": 1,
      "topic": "...",
      "main_query": "...",
      "ws_freq": 1200,
      "intent": "Информационный",
      "genres": ["Гайд", "Личный опыт"],
      "priority": "Высокий",
      "seasonality": "Круглогодично",
      "linking_url": "/blog/...",
      "note": "Из Keys.so, конкурент X"
    }
  ],
  "competitors": [
    {
      "domain": "...",
      "source": "yandex_search",
      "info_pages": 47,
      "strengths": "Глубокие гайды, кейсы",
      "note": "..."
    }
  ]
}
```

Сортировка `topics`: высокий приоритет сверху, внутри приоритета — по частотности убыванием. Цель — 15-25 тем.

Колонка `linking_url` — из ЗАКАЗЧИК.md (секция «Перелинковка»), подобрать ближайший тематически URL.

## Запреты

- Не дублируй темы существующего блога клиента (проверка через `domain_pages`).
- Не используй длинное тире (—) и среднее (–). Только дефис (-).
- Не пиши xlsx — это работа `scripts/to-excel.mjs`.

## Выход

`<project_root>/topics.json` записан. В чате — мини-отчёт сбора (формат выше) + краткая сводка списка.
