# structure-writer

Твоя задача - собрать финальный артефакт **A6.md** - структура сайта в markdown. Этот файл идёт в проект и используется в последующих скилах (У5 ТЗ верстальщика, У6 контент-план, /seo-statya для приоритизации страниц).

MCP не используешь. Это чистая компиляция данных в markdown по фиксированному формату.

## Вход

- `structure_dir` - путь к `structures/NNN-<slug>/`
- `analysis_dir` - путь к `analyses/NNN-<slug>/`
- `project_root` - корень проекта

## Обязательное чтение

1. `<structure_dir>/structure_data.json` - **главный источник**: правленая клиентом версия мастер-списка с полем `target_status` (`yes` / `no` / `discuss`-разрешённое), `section` (раздел/хаб меню, если структура сгруппирована) и `category` (третий уровень для товарных сайтов; для услуг обычно пусто).
2. `<structure_dir>/cannibalization.json` - `recommendations[]` (рекомендации по расширению), `resolutions[]`.
3. `<structure_dir>/master_list.json` - `pairing_performed`, `unique_features`, миграционные решения (для раздела «Миграция»), а также:
   - `use_sections` (bool) - включена ли группировка (true почти всегда; см. ниже).
   - `sections[]` - определения разделов/хабов верхнего уровня меню (`id`, `name`, `axis`, `note`).
   - `competitor_url_depth` (object) - замер вложенности URL конкурентов: `median_segments`, `dominant_pattern` (`flat`/`one_level`/`two_level`/`deep`), `examples[]`, `note`.
   - `url_nesting_recommendation` (object) - политика по URL: `mode` (`flat`/`nested`), `rationale`, `migration_needed` (читается в разделе «Миграция»).
   - Все новые поля парси с guard на отсутствие (как существующий гейт по `use_sections`): нет поля - веди себя как при дефолте (`category` пусто, `mode` = `flat`).
4. `<structure_dir>/inputs.json` - параметры проекта (`slug`, `domain`, `keyso_base`, `region_yandex`, `region_name`, `note_region`, `analysis_reconstructed`).
5. `<analysis_dir>/brief.json` - `niche`, `region`, `domain`, `company_name`.
6. `<analysis_dir>/competitors.json` - `direct[]`, `leaders_top3[]` для секции «Конкуренты».
7. `<analysis_dir>/A3.md` - стоп-лист (для информации в шапке, опц.).
8. `<structure_dir>/semantic_pack.json` (опц.) - поля `degraded`, `degraded_reason`, `region_note` (для блока «Замечания прогона» - честно отразить, если JM деградировал).
9. `<structure_dir>/decisions.json` (опц.) - журнал авто-решений алгоритма (для раздела «Наши SEO-решения»; low-confidence выносятся отдельным чек-листом).

Запиши `<structure_dir>/A6.md` со следующей структурой. **Жёстко соблюдай порядок разделов и подразделов.**

````markdown

**Источник:** /seo-struktura NNN (на базе /seo-analiz NNN)
**Аналитик:** TIMUR SEO

## Блок перелинковки в шапке

> ТЗ на сквозной блок перелинковки в шапке сайта (это спецификация, не HTML-разметка). Какие сгруппированные страницы кросс-линкуются в шапке, чтобы передавать вес и навигацию между смежными разделами/категориями.

<Если `master_list.use_sections == true`:>

- **Что линкуем:** в шапке - сквозное меню по разделам из `master_list.sections[]` (верхний уровень); под каждым разделом - его целевые страницы (`target_status == "yes"`, `section` совпадает). Для товарных сайтов промежуточный уровень - категории (`category`), под ними товары.
- **Состав по каждому разделу** (перечисли построчно тот же набор, что в «Архитектура меню (шапка)»):
  - **<section.name>:** <Название - URL>, <Название - URL>, ... (для товаров - сгруппировано по `category`)
- **Принцип перелинковки:** каждая сгруппированная страница доступна из шапки на всех страницах сайта; страницы одного раздела (а для товаров - одной категории) образуют связанный кластер. Страницы «Вне разделов» (Главная, контакты) - в общей части шапки, без привязки к кластеру.
- **Анкоры:** человекочитаемое название страницы (из `structure_data`), без переспама ключами.

<Если `master_list.use_sections == false`:>

Группировки нет (плоский мелкий набор) - отдельный блок перелинковки по разделам не требуется; все целевые страницы линкуются из общего меню шапки.
````

## Правила формирования

### URL (ЧПУ) для новых страниц

Если `migration_decision == "new"` и URL не указан клиентом - сгенерируй ЧПУ:
- Транслитерация с русского
- Lowercase
- Дефисы вместо пробелов
- По правилам типа: услуги -> `/uslugi/<slug>/`, категории -> `/catalog/<slug>/`, статьи -> `/blog/<slug>/`, инфо -> `/<slug>/`
- Примеры:
  - «Ремонт ванной комнаты» (service) -> `/uslugi/remont-vannoj-komnaty/`
  - «Холодильники Bosch» (category) -> `/catalog/holodilniki-bosch/`

### Приоритет (поле в таблице целевых)

- **высокий** - `coverage_pct >= 50` ИЛИ `marker_freq >= 500`
- **средний** - `coverage_pct` 25-49 ИЛИ `marker_freq` 100-499
- **низкий** - `coverage_pct < 25` ИЛИ `marker_freq < 100` ИЛИ `source == "brief"` без подтверждения частотности

### Статус (поле в таблице целевых)

- `existing` (если `migration_decision == "existing"`) - оставить как есть
- `redirect` (если `redirect_301`) - 301 редирект
- `обсудить` (если `migration_decision == "discuss"`) - решение по странице не принято
- `к удалению` (если `migration_decision == "delete_410"`) - страница убирается (410)
- `new` - новая страница

> `target_status` к этому шагу нормализован скилом до `yes`/`no` (гейт). Если в данных встретилось неопределённое значение (`empty`/`discuss`) - выведи страницу в «Отложенные» с пометкой «статус не определён», не теряй её молча.

### Если страница имеет `marker == null` (информационная или унаследованная)

В таблице целевых: маркер и все запросы - прочерк (-). Тип = `info` или `product` соответственно.

### Жанр текста

- Деловой, точный, без воды.
- **Длинные тире (—) и средние (–) не использовать. Только дефис (-).**
- Все цифры - из JSON-источников. Никаких выдуманных метрик.
- Если поле пустое - пиши «не заполнено» / «не указано», не оставляй пустую строку.

- Распределение по типам (целевые): home `<X>`, category `<X>`, service `<X>`, product `<X>`, article `<X>`, info `<X>`
- Приоритеты (целевые): высокий `<H>`, средний `<S>`, низкий `<L>`

- Политика URL: <«плоские, группировка только в шапке» / «вложенные /razdel/usluga + 301-миграция»> (по `url_nesting_recommendation.mode`)
- Миграция: <X существующих, Y 301-редиректов, Z удалить, W обсудить> или «не выполнялась»
