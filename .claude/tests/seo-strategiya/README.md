# Smoke-тесты `/seo-strategiya` - verify-strategy.mjs

Регрессионные тесты для механического гейта стратегии (шаг 6.5а, между шагом 6
"Контент стратегии" и шагом 7 "Сборка docx"):
- `verify-strategy.mjs` - проверка `seo-strategiya_content.json` на цены в прозе
  тарифов, стоп-паттерны воды, тире/букву Е-с-точками, грубый объем (warning),
  санити-структуру.

## Что проверяют (9 кейсов, 11 шагов - тире и буква Е-с-точками разнесены на два шага)

| # | Тест | Что валидируется |
|---|---|---|
| 1 | чистый контент | exit 0 на валидном документе (6 разделов, 3 тарифа без цен, проза без стоп-слов/тире/буквы Е-с-точками) |
| 2 | цена с валютой в hint тарифа | exit 2, заголовок «ЦЕНЫ В ПРОЗЕ ТАРИФОВ», фрагмент с суммой в выводе |
| 3 | круглая тысяча без валюты в `services[].description` | exit 2 (мягкий флаг круглой тысячи в секции 4 - тоже нарушение) |
| 4 | легитимные ₽ в секции 6 (декомпозиция выручки) | exit 0 - ценовой скан НЕ трогает секцию 6, даже с "25 000 ₽" в тексте |
| 5 | стоп-паттерн «комплексный подход» в `paragraph.text` | exit 2, заголовок «СТОП-ПАТТЕРН...» |
| 6 | длинное тире (—) в прозе | exit 2, заголовок «ТИРЕ/Е-С-ТОЧКАМИ» |
| 6б | буква Е-с-точками в `title_page.author` (служебное поле) | exit 2 - служебные поля тоже проверяются, не только раздел-проза |
| 7 | нет раздела с `id="4"` | exit 2, заголовок «СТРУКТУРА» - без вариантов работы docx бессмысленен |
| 8 | отсутствующий `seo-strategiya_content.json` | exit 1 |
| 8б | битый JSON (невалидный синтаксис) | exit 1 |
| 9 | тонкая проза (< 3500 симв.), но чистая | exit 0, заголовок «ОБЪЕМ» с предупреждением - объем НЕ блокирует |

## Как запустить

Из корня проекта:
```
.claude\scripts\_node.cmd .claude\tests\seo-strategiya\run.mjs
```
Exit 0 = все ок. Exit 1 = хоть один тест упал (вывод покажет где).

Фикстуры генерируются прямо в `run.mjs` (в песочнице `.claude/tmp/seo-strategiya-test`),
внешних файлов нет - синтезируются inline через `cleanContent()` + точечные мутации
на кейс (по образцу `tests/metatags/run.mjs`).

## Когда запускать

- После правок в `verify-strategy.mjs`.
- После правок стоп-листа воды в `strategy-writer.md` (стоп-лист скрипта - дословная
  копия, синхронизировать вручную при расхождении).
- Перед PR / push.

## Что НЕ покрыто (требует живого opus, тестируется вручную/агентом)

- Смысловая сверка цифр стратегии с `seo-strategiya_data.json` (DR/ИКС/ТОП-10/
  трафик/конкуренты/forecast/decomposition) - это `strategy-verifier` (Пакет B).
- Согласованность тарифов в прозе с `tariffs.json` (состав услуг, рекомендованный
  вариант) - тоже `strategy-verifier`.
- Суждение об объеме "на глаз" (сколько реально страниц выйдет с учетом таблиц) -
  скрипт дает только грубый warning по числу символов прозы, точную оценку дает
  `strategy-verifier`.
- Анти-вода сверх фиксированного стоп-листа (канцелярит, пустые вводные без факта) -
  тоже `strategy-verifier`.
