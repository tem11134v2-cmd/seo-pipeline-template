# tests/style - регресс стилевых правил (буква ё / тире)

Проект запрещает букву **ё** во всех клиентских текстах и метатегах (наравне с
длинным/средним тире): пишем всегда **е**. Правило держится на двух рубежах, и
этот набор ловит регресс, если один из них уберут.

## Запуск

```
.claude\scripts\_node.cmd .claude\tests\style\run.mjs
```

Exit 0 - всё зелёное (песочница `.claude/tmp/style-test` удаляется за собой).
Exit 1 - есть провал (песочница остаётся для разбора).

## Что проверяется

1. **Функциональные (блокировка на ё).** Синтетические фикстуры с буквой ё:
   - `verify-article-metatags.mjs` (метатеги статьи) -> exit 2;
   - `verify-copy.mjs` (копи страницы) -> exit 2;
   - `hooks/check-section.sh` (раздел статьи) -> exit 2. Требует sh/bash; при её
     отсутствии шаги помечаются SKIP и не валят набор.
2. **Контроль обратного.** Те же фикстуры без ё и без тире проходят те же
   проверки без нарушений (exit 0) - нет ложных срабатываний. Пары clean/yo
   различаются ровно одной буквой (надежно/надёжно).
3. **Дрейф-гарды.** Дёшево, без запуска: паттерн `[ёЁ]` присутствует в
   verify-скриптах, а нормализация `/ё/g` (ё->е) - в сборщиках docx/html/md.
   Ловит «тихое» удаление проверки или нормализации.

Набор герметичен: все фикстуры генерируются во временную папку и удаляются при
успехе; в рабочих директориях проекта ничего не остаётся.
